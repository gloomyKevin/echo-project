# 随 · Proto

交互原型，用于验证核心滑动→打卡→庆祝链路与本地推荐逻辑。

---

## 启动

```bash
npm install   # 首次
npm run dev   # 访问 http://localhost:5173
```

---

## 三个屏幕

| 屏幕 | 触发方式 |
|------|---------|
| **发现页** — 卡片栈 | 默认页面 |
| **打卡弹窗** | 点击卡片底部「已经做过 →」 |
| **庆祝页** | 打卡提交后 |

---

## 核心手势

| 手势 | 效果 |
|------|------|
| 右滑 / 拖过中线 | 感兴趣，记入偏好权重 |
| 左滑 / 拖过中线 | 没兴趣，暂时排除该卡片 |
| 点击「已经做过 →」 | 进入打卡流程 |

---

## Dev Panel 使用

点击右下角 **🛠** 按钮展开，三个 tab：

### 聚类 tab
- **需要至少 5 次滑动**才开始显示有意义的信号
- `独处 ↔ 社交`、`室内 ↔ 户外` 偏好条：条越靠右代表越偏右侧
- 域偏好柱状图：显示每个域的右滑率（reflective / exploration / social / physical / creative / food）

### 维度 tab
- 6 个生活维度的当前分数（初始值均为 50）
- **只有完成打卡才会推高分数**，右滑不影响分数

### 推荐 tab
- 当前推荐栈前 8 张，显示算法得分和得分理由
- 格式：`域匹配 XX% | 场景匹配 XX% | 强度适配 ×1.0 | 🃏野卡`

---

## 推荐引擎逻辑（可验证点）

```
1. 硬过滤   → 已拒绝 / 已完成的卡片不再出现
2. 软排序   → 右滑越多的域，同域卡片得分越高
3. 强度渐进 → 打卡 < 3 次：只推 low；3–7 次：引入 medium；≥ 8 次：引入 high
4. 野卡注入 → 每第 5 张插入一张低偏好域卡片（标注 🃏野卡）
```

**验证方法：**
- 连续右滑社交类卡片 → 「推荐」tab 社交域卡片排名上升
- 完成 3 次打卡 → medium 强度卡出现在推荐栈
- 数一数每第 5 张是否带 🃏野卡标记

## 测试路径建议

### 路径 A — 快速验证链路（5 分钟）
1. 右滑 3 张，左滑 2 张
2. 点「已经做过」→ 选情绪 → 提交
3. 确认庆祝页维度条有变化
4. 回到发现页，开 Dev Panel 查看打卡计数变为 1

### 路径 B — 验证偏好学习（10 分钟）
1. 打开 Dev Panel → 聚类 tab
2. 连续右滑 5 张以上，注意聚类信号开始变化
3. 对「探索」类卡片（户外、换路回家等）全部右滑
4. 查看推荐 tab，探索域卡片得分应明显高于其他域
5. 观察第 5 张推荐是否带 🃏野卡标记

### 路径 C — 验证强度渐进（完整流程）
1. 连续完成 3 次打卡（Dev Panel 打卡计数 ≥ 3）
2. 查看推荐 tab，medium 强度卡片（日常·中强度）开始出现
3. 完成 8 次打卡后，high 强度卡（大事·高强度）出现

---

## 重置数据

Dev Panel 底部「重置数据」按钮 → 清空 localStorage 所有状态，重新冷启动。

---

## 已知限制（Phase 0 范围内）

- 无账号、无同步，数据仅存本机 localStorage
- 卡片池固定 20 张，刷完后提示「去做一件事，回来继续」
- 强度渐进阈值（3 / 8 次打卡）为固定值，Phase 1 可依数据调整
- Dev Panel 仅在本地开发时使用，不面向真实用户
