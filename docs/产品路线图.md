## Phase 0：Web Demo

**定位**：几天时间，只为验证核心交互感受是否对，不是正式产品。

**做什么**：
- 卡片滑动交互（右滑 / 左滑 / 上滑）
- 打卡流程（照片/跳过 + 情绪贴纸）
- 六维分数变化可见
- 全部数据在本地（刷新后重置是可以接受的）

**不做什么**：登录、账号、后端、地图、推荐算法、碎片生成。这些 Phase 1 再说。

**技术方案**：Expo Web → `vercel deploy` → 得到一个 https 链接直接分享。卡片数据写死在 JSON 文件里。Phase 0 的代码直接复用到 Phase 1 的 Expo 工程，不浪费。

---

## Phase 1：iOS TestFlight

**新增**：
- 真实账号体系（Apple Sign In + 邮箱）
- 数据上云（Supabase），换手机不丢
- 卡片内容可后台更新（不需要发版）
- 基础分析（Amplitude）
- 推送通知（第一次打卡完成后申请权限）

**发布节奏**：
1. 内部测试先跑（≤100台，无需 Apple 审核）→ 发给第一批 10-30 人
2. 提交外部测试申请（Apple 审核 1-2 天）→ 扩至更大社群
3. 两件事并行推进，不互相阻塞

**Android**：Expo 同一套代码可以同时输出 Android APK，成本极低。Phase 1 期间内部安装自测，不对外分发，等 iOS 验证完再决定是否同步推进 Android。

## Phase 2+：App Store 正式发布

**前置条件**：Phase 1 中核心指标达标（D7 留存 >40%，打卡转化 >30%）。

**新增**：
- 城市探索地图（Phase 2 首要功能）
- 碎片分享图生成
- AI 预演文字（batch 离线生成，先跑热门卡片）
- 逆向 Onboarding

**Phase 3+（不排期，按需推进）**：
- Android 正式发布
- 季节性内容运营
- 先行者机制择机放出

## 基础设施决策

### 公司注册与服务访问

**建议在新加坡注册实体**，理由：

- 直接访问所有全球服务（Claude API、Stripe、App Store 全区、Supabase 等），不需要代理
- 新加坡离大陆近（网络延迟约 40-60ms，对这类 App 完全可接受）
- 法规清晰，对国际化友好，是中国出海的主流选择
- HK 也可行，但近几年全球接受度不如新加坡稳定

> **关于「代理」**：本质问题是大陆境内的服务器访问 Anthropic 等境外 API 受网络管控影响。解决方案很简单：把所有后端部署在新加坡境外服务器（Vercel/Supabase），大陆用户连接你的新加坡后端，后端再调用 Claude API——全程合法，延迟可接受，不需要任何代理工具。注册实体在新加坡是这个方案的前提。

**国内 App Store 发布**：Apple Developer 账号本身不限区域，TestFlight 和 App Store 国区发布均可用全球账号操作。（若未来涉及国区内购付款，需要单独了解苹果与国内支付渠道的对接，现阶段不需要考虑。）

### 后端地理架构

```
大陆用户
   │  （直连，~50ms）
   ▼
Vercel Edge + Supabase（新加坡节点）
   │
   ├── Claude API（Anthropic，全球访问）
   ├── 高德地图 API（国内地图数据）
   └── Mapbox（全球地图数据，Phase 2+）
```

地图说明：高德提供精确的国内地图数据（国内法规要求，坐标系与全球有偏移），Mapbox 覆盖全球。Phase 2 做地图功能时，根据用户 IP/手机 locale 自动切换，上层代码用统一的地图抽象层封装，不暴露切换逻辑。

## 技术栈清单（已决策）

| 层 | 选型 | 说明 |
|----|------|------|
| 客户端框架 | Expo (React Native) | iOS + Android + Web 同一套代码 |
| 动画/手势 | Reanimated 3 + Gesture Handler | 卡片滑动必须从 Day 1 用对，不能后补 |
| 状态管理 | Zustand + persist middleware | 本地/云无缝切换 |
| Web 托管 | Vercel | Phase 0 demo + 未来 Web 端 |
| 后端即服务 | Supabase（新加坡节点） | Auth + DB + Storage，Phase 1 不需要自己写服务器 |
| 移动端 CI/CD | EAS Build | TestFlight 一条命令 |
| AI | Claude API（Anthropic） | Phase 2+ batch 生成，Phase 1 暂不接入 |
| 分析 | Amplitude 免费版 | Day 1 接入 |
| 崩溃监控 | Sentry 免费版 | Phase 1 接入 |
| 地图（Phase 2+） | 高德（国内）/ Mapbox（全球） | 统一抽象层封装，上层代码不感知差异 |
| 推送 | Expo Push Notifications（封装 APNs） | Phase 1 接入 |
