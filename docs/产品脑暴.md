# 产品头脑风暴

## 一句话描述

帮你发现生活的可能性，推动你去体验，在每一次发生中认识自己。

## 出发点

AI 革命后，世界越来越丰富，生活应该越来越充实而不是越来越贫瘠。人应该更多建立和世界的连接，更多跳出舒适区——而不是被包裹在信息茧房变得极端，不是完全沉沦于线上，不是自叹于世界的割裂。

## 核心理念

**认识自己 → 发现可能 → 发生即奇迹**

先帮你看清自己是谁，再帮你看见生活还能怎么过，然后推你一把去体验——每一次发生都是奇迹。

未来拓展方向：找到同好、共同分享（不是现阶段重点）

## 产品设计原则

- **Don't make me think** —— 懒人式、选择式为主，不要让用户思考
- **如无必要，勿增实体** —— 克制，不堆功能
- **Social Proof（社会认同）** —— 让用户看到别人的选择和体验，降低决策门槛

---

## 愿景分层

### 第一阶段：发现与体验（当前聚焦）
- 做一个交互创新的生活可能性发现工具
- 通过卡片池让用户看到"原来我还可以这样"
- 核心价值：不是替你做决定，而是帮你发现你不知道自己想要的东西
- 通过每一次交互，自然沉淀用户的偏好、价值观、性格数据

### 第二阶段：人格数据化引擎（下一步）
- 积累足够数据后，训练/微调模型
- 构建用户的「数字人格档案」——性格、偏好、价值排序、决策模式
- 从"帮你发现可能"进化到"真正懂你的 agent"

### 第三阶段：通用人格规范（终极野心）
- 把「数字人格档案」做成开放规范/协议
- 任何 AI 产品都能一键导入这份存档
- 用户换一个 AI 不用从零开始，所有 AI 都能秒懂你
- 你的人格数据属于你自己，可迁移、可授权、可撤回

## 原子行为体系

不管是代码实现、模型训练还是自然语言到选择的映射，都需要一套自建的、可以替代人的自然行为或意识的原子体系。

### 原子维度（举例）
- **时间**：不加班的周末、清明假期、说走就走……
- **财务**：这个月月光了、有闲钱、预算有限……
- **事件**：看电影、旅游、拜访朋友、买东西……
- **舒适区突破**：恐怖片、没去过的城市、没试过的食物……
- **导向结果**：能量恢复、社交感情增加、视野拓展……
- **外部状态**：事事不顺、信命、想改运势……

### 原子组合示例
- 周末 + 不加班 + 月光 → 看电影 → 挑战恐怖片 → 看完找朋友聊 → 能量恢复 + 社交↑
- 清明假期 + 不考试 + 说走就走 + 爱好旅游 → 去清迈
- 事事不顺 + 信命 + 改运势 → 换城市 → 城市选择卡片

### 技术策略
- 前期没有 token 资源时，先做离线枚举和预设
- AI 离线缓存一批原子数据
- 后续逐步接入 AI 实时生成

## 卡片系统（核心交互）

### 卡片类型与层级

**从微粒卡到主线卡的梯度：**
- **微粒卡**："今天上床不带手机"、"买一杯没试过味道的奶茶"
- **日常卡**：看一部电影、去一家新餐厅
- **事件卡**：周末露营、短途旅行
- **大爆炸卡**：如"不要错过春天"——包含多个子卡片项，AI 给出连贯的事件安排建议
- **围观卡**：有趣的尝试，如"高中生特种兵游玩新加坡"——选择尝试后，AI 基于你的现状做实现路径拆解
- **抽象里程碑卡**：高考、考研、毕业等人生大节点——独立路径 + 点亮 + 彼此留言鼓励

### 卡片内部机制
- **丰富度分级**：内容覆盖广度
- **置信度分级**：推荐把握程度
- **可执行度分级**：基于用户当前状态是否真的能做
- **红线机制**：不能推荐的边界
- **层层筛选**：从生成到呈现的过滤管道
- 前期尽可能丰富，讨好用户

### 卡片交互创意
- 首页呈现形式待深挖（信息流？卡片墙？）
- 左右滑选择
- 跳出舒适区类型 → 投骰子/随机选择，降低心理门槛
- 卡片翻转可看热度、留言、鼓励
- 前期最小锚点："今天上床不带手机"、"买一杯没试过的奶茶"这类微行动

### 选择的预演
在选择做出之前，AI 基于对世界/经验的了解，给出：
- 过程预演：中间会经历什么
- 结果预演：可预期的结果
- 指标变化：六维度量衡的预期变动

### 完成后的体验
- 完成卡片后，选择极简贴纸表达感受：治愈、开心、释然、疲惫、突破、平淡
- 生成专属编号的碎片卡 + 微光特效
- 点亮时触发轻微震动 + 微光动画
- **每一次行动都有专属记忆点**

## 六维度量衡

所有数据基本收束在六个指标中：

| 维度 | 含义 |
|------|------|
| 健康 | 身体与心理状态 |
| 财务 | 经济状况与消费 |
| 事业 | 工作、学业、成长 |
| 社交 | 人际关系与连接 |
| 环境 | 生活环境与物理世界 |
| 可能性 | 视野拓展、舒适区突破 |

### 数据采集方式
- **绝不做 MBTI 等问卷测试**
- 通过前期自然的选择、长期的沉淀，自动默默得出对你的了解
- 每次卡片交互、选择、完成、反馈都在自然采集

### 第一个里程碑：人生蓝图
数据积累到一定量后，生成你的人生规划蓝图说明：
- 基础情况
- 日常拆解
- 社交圈
- 爱好与讨厌
- 尝试过的努力
- 当下的焦虑
- 短期目标、长期目标

生成后可精细调整，定下人生里程碑后，自动将大需求拆解为小事情，陪伴、提醒、不断给正反馈。

## 冷启动悖论与推荐演进

### 核心矛盾

推荐质量依赖用户画像，用户画像依赖推荐交互数据的积累——典型的鸡生蛋问题。解法不是等数据积累，而是设计一条**从零数据到精准推荐的演进路径**，每个阶段都有不同的推荐策略，同时对用户构建正确的心智预期。

---

### 真正的冷启动：纯本地、无 AI、少量用户

在 Phase 0 demo 阶段（静态卡片库 + localStorage，给最熟悉的几十个朋友体验），个性化推荐根本不存在——也不需要存在。这个阶段真正要解决的不是推荐精准度问题，而是两个更根本的问题：

**问题一：在没有任何个人数据的情况下，如何让第一批用户感觉「这不是随机推给我的」？**

解法：**编辑即算法**。在数据驱动的个性化到来之前，高质量的人工策划本身就是最好的推荐。好的书店不了解你，但它的选品让你觉得「这里的东西不是乱摆的」。

具体做法是把卡片的展示顺序作为一种设计决策，而不是随机或按创建时间排列：

```
首屏前 3 张：普世触点——几乎所有人都会觉得有点共鸣
（「今晚上床不带手机」「给很久没联系的人发条消息」）
第 4-10 张：最大化域覆盖——探索/内省/社交/身体各出现一次
            目的是收集信号，看用户在哪个方向停留更久
第 11-15 张：开始轻微分叉——根据前几张的滑动速度调整
第 16+：引入「野卡」——和用户已有偏向完全不同的方向，防止窄化
```

这个排序的哲学：**前几张负责建立信任，中间几张负责采集信号，后面几张开始分叉。** 用户感受到的是「推得挺准」，背后是精心设计的展示顺序而不是算法。

**问题二：卡片本身的可信度和可执行度，在没有任何用户数据背书的情况下如何建立？**

这个问题比推荐悖论更早发生。用户看到一张卡片，心里的第一个问题不是「这是不是为我推的」，而是：

> 「这真的值得我去做吗？做了我会后悔吗？」

解决这个问题有三个层次：

**层次一：卡片设计本身建立可信度（不依赖任何数据）**

可信度不来自「N 人做过」的数字，而来自预演文字的质地。用户能感觉出来一段文字是「真的做过才知道」还是「想象出来的描述」：

- 「头十分钟可能有点不习惯，想去摸它」—— 感觉真实，写这个的人做过这件事
- 「手机不在身边会让你更加专注和放松」—— 感觉是在告诉你结论，没有经历感

关键不是文字有多优美，而是**足够具体、足够诚实、包含意外**。尤其是「意外」——告诉用户一件他们做之前不会想到的事，这是建立预演可信度最有效的手段。

**层次二：可执行度的基础筛选（卡片元数据层面）**

每张卡片有明确的时间/预算/地点要求。在展示之前先做基础可行性粗筛：

```
工作日晚上 22:00 打开 App → 时间要求 ≥ 2 小时的卡片降权
当前在家 → locationRequired = 'city' 的卡片降权
```

这不是个性化推荐，是最基本的「别推我做不到的事」。做到这一点，用户对 App 的基础信任就建立了。

**层次三：让用户自己评估（不替用户判断）**

「了解一下」按钮展开的预演，就是把可执行度评估权交给用户的设计。App 不说「你现在应该去做这个」，它说「这件事大概是这样的，你来判断」。这个信任转移是关键——特别是在产品没有用户数据的阶段，假装能判断用户是否适合做某事，会立刻被识破。

### 三阶段推荐演进模型

#### Phase 0 — 上下文驱动（首次开启，零用户数据）

不问用户任何问题。仅用客观可获取的信号：

| 信号 | 获取方式 | 用途 |
|------|----------|------|
| 当前时间 | 系统 | 22:00 工作日 → 排除需要规划的事件卡 |
| 星期几 | 系统 | 工作日 vs 休息日 → 完全不同的卡片池 |
| 地点类型 | GPS（需授权） | 在家/通勤中/外出 → 对可执行度做粗筛 |
| 即将到来的假期 | 系统日历 | 节前 3 天 → 推出游/旅行相关卡片 |
| 设备使用时段 | 行为 | 深夜常开 = 夜猫生活方式，早起 = 不同节奏 |

仅凭这些，就能做出「及格的推荐」：正确的时间，推正确规模的卡片。

**首屏味道采样（Taste Seeding）**

首次进入 App，在正式刷卡之前：展示 15 张风格差异极大的概念卡（只有图片和一句话），用户做快速滑动（2-3秒每张），全程约 90 秒，拿到初始偏好向量。

这不是「选择兴趣标签」——没有分类，没有表单，只是快速感性反应。参考 Pinterest 的 onboarding 视觉板选择，但更感性、更快。完成后用户不知道自己做了「测试」，但系统拿到了足够起步的方向信号。

---

#### Phase 1 — 聚类引导（Day 1-14，20-30次交互后）

达到最小数据量后，将用户软归入行为聚类。**聚类不是人格标签，是行为模式的描述。**

建议的聚类维度交叉：
- 独处偏好型 / 社交驱动型
- 向内探索型（内省、创造、反思）/ 向外行动型（探索、身体、冒险）
- 低强度舒适型 / 高刺激挑战型

软归属原则：用户可以是 70% 独处探索者 + 30% 感官冒险家，聚类之间的权重随交互不断修正。

聚类的价值：用「相似的人做过什么」来填充个人数据的空缺——这是经典的协同过滤，在用户个人数据不足时非常有效。

---

#### Phase 2 — 个人信号主导（Week 3+）

当个人数据密度超过聚类均值后，完全切换到个人推荐。聚类保留一个作用：**野卡注入来源**——从用户所在聚类以外的内容注入，保持发现感，防止过度个性化导致的信息茧房。

---

### 可无需询问推断的信号（系统感知层）

这些信号比任何问卷都诚实，因为来自真实行为：

**行为信号**
- 滑动速度：快速左滑 = 明确不感兴趣；慢悠悠才左滑 = 犹豫，可能只是时机不对
- 滑动时间段：早晨刷卡 = 计划模式（正在考虑做什么）；深夜刷卡 = 幻想模式（在想象而非计划）
- 开 App 频率：每天多次 vs 每几天一次 = 截然不同的参与度状态
- 在某张卡片上的停留时间：超过正常阅读时长 = 有兴趣但犹豫

**上下文信号（授权后）**
- 位置模式：家/通勤/工作区/陌生区域
- 日历状态：有没有被占满的周末，即将到来的长假，最近完成了大事

---

### 用户心智建设：aha moment 工程

冷启动期最大的风险不是推荐不准，而是用户在「准了」之前就离开了。必须主动工程化「aha moment」的到来时间：

**Session 1 目标**：至少一张卡片让用户觉得「这个我真的想做」（用聚类的高置信推荐，用熟悉触发感兴趣）

**Session 2 目标**：至少一张卡片让用户觉得「它怎么知道的」（开始引入略微意外但精准的推荐，制造惊喜感）

**Session 3 目标**：系统第一次反馈它学到了什么（「你最近对独自探索的事更感兴趣，是吗？」——可纠错的透明）

**透明化策略（关键）**

用户需要知道系统是如何了解他们的，否则精准推荐会感觉「creepy」。

- Onboarding 第一句话不是介绍功能，是说哲学：「Vibe 不问你喜欢什么——它通过看你真正做什么来了解你。你每一次选择，都在教它。」
- 不用进度条，用隐喻：种子/光点图标，代表系统对你的了解程度，只有三个阶段（刚认识/开始了解/比较懂你）
- 第一次打卡后：「你做了第一件事。Vibe 开始认识你了。」
- 约 20 次交互后：首次透明片段：「根据你目前的选择，你似乎更倾向于独处的体验。这对吗？」+ 可调整入口
- 这种「可纠错的透明」建立信任，避免黑箱感

**行业参考**
- **Netflix**：用 genre 行为归类，而非填写问卷表单——我们的「滑动即采集」是同样逻辑
- **TikTok**：完播率比点赞更诚实，极速收敛——我们的「打卡」比「右滑」更有信号价值，打卡一次抵右滑十次
- **Spotify**：用客观音频特征（BPM、能量值）作为不依赖用户数据的推荐维度——我们卡片的原子元数据（时间、预算、社交维度等）扮演同等角色
- **Duolingo**：渐进式承诺（选语言 → 设目标 → 开始），每步都很小但建立了投入感

## 反馈与追踪机制的设计哲学

### 核心命题

这里有一个产品价值观层面的根本选择，需要先定清楚：

> 用户是在「向 App 汇报」，还是在「见证自己的生活」？

前者产生工具性压力，后者产生意义感和主动性。Vibe 的所有反馈设计都应该服务于后者。

设计不是功能的堆叠，而是一套最小必要的闭环，让体验变成记忆，让记忆变成认识自己的素材。

---

### 三个反馈触发时机（不多，不少）

**① 再次打开 App 时的「回声」（最自然的时机）**

用户上次右滑了某张卡，下次打开 App 时，首屏静默一秒后，轻轻出现：

> 「上次你想去做 [卡片标题]，去了吗？」
> [去了，记一下] / [还没] / [取消]

关键：这不是推送通知，是 App 内的自然时机，像一个朋友问一句「怎么样了」。摩擦极小，不打断用户想做的事，只在 App 开启的一瞬间轻触。

**② 时间触发的极简推送（仅两次，不重复）**

- 右滑后第 3 天：一条推送，只有卡片标题 + 一个箭头。没有催促词。
- 第 7 天：同样极简，文案轻微调整：「还有时间的。」
- 第 7 天后：停止推送。卡片开始自然褪色。不再追。

两次的逻辑：第一次是提醒，第二次是轻推，第三次就是骚扰。

**③ 地点触发（仅对位置绑定卡，需权限）**

检测到用户进入已收藏的位置绑定卡的地点附近：
> 「你现在正好在 [地点名] 附近，要做这件事吗？」

这是最有仪式感的触发方式，因为它是真实世界和 App 世界在这一刻对齐了。发生频率低，但每次都很有价值。

---

### 打卡的「够了」边界

产品不说「你必须做什么」——通过设计传递「这就够了」：

```
情绪贴纸一个 = 完整的打卡
```

没有必填的照片，没有必填的文字，没有字数要求。情绪贴纸是唯一的必选项，因为它是最有数据价值的信号（比文字更结构化，比评分更人性化），也是最低的表达门槛。

其他所有内容（照片、文字）都是可选的自我丰富，不是义务。

**「够了」在设计上如何体现：**
- 打卡流程中，选完情绪贴纸后，「完成」按钮立刻可点，不等其他任何输入
- 文字输入框的 placeholder 是「可以写一句话，也可以不写」，而不是「说说你的感受吧」（后者暗示了义务）
- 没有字数提示，没有进度条

---

### 静默完成者的处理

有些用户会默默去做体验，但不打卡。这是正常的，不应该被惩罚，也不应该被忽略。

设计原则：**接受不可见的完成，留一个补记的入口**。

- 对于静默完成者，不催促、不降低评分、不减少推荐质量
- 任何卡片都可以通过「上滑」标记「已经做过了」，并可以补充情绪贴纸，哪怕是三个月前的事
- 界面里有一个很安静的入口：「那些你做了但没记录的事，也可以在这里留下来。」

---

### 明确不做的设计

这些在同类产品里很常见，但和 Vibe 的核心价值观冲突：

| 不做什么 | 为什么不做 |
|----------|-----------|
| 连续打卡 Streak | 把「不中断」变成目标，产生错误的焦虑驱动 |
| XP / 等级系统 | App 变成目的而非工具 |
| 完成率数字显示 | 低完成率会变成负担而非激励 |
| 以打卡频率换徽章 | 诱导记录行为而非体验行为 |
| 每日推送提醒「去体验吧」 | 变成噪音，用户会关掉通知 |

## 选择预演的科学性

### 核心问题

选择预演是本产品最有可能成为差异化灵魂的交互——但也是目前最空洞的部分。预演文字如果是编辑直觉产物，会在第一次不准的时候失去用户信任。需要建立一套**可以随时间演进、有理论基础、能自我校正**的预演生成体系。

---

### 三重科学来源

#### 来源 1：心理学研究先验（冷启动期的基础）

六个维度各有对应的扎实研究领域，这些研究为每类行动原子设置初始维度影响区间：

| 维度 | 对应研究领域 | 典型结论 |
|------|------------|---------|
| 健康 | 运动心理学、压力恢复研究 | 哪怕一次中等强度的有氧运动，当天情绪显著改善（效应量大） |
| 社交 | Cacioppo 孤独与健康研究 | 一次真实的社交互动对心理健康的影响往往当天可测 |
| 环境 | 环境心理学、自然暴露研究 | 在自然环境中停留 20 分钟，压力激素可测量下降 |
| 可能性 | Aron 的自我扩展理论 | 接触「新异体验」会短期内扩大自我概念范围 |
| 事业 | Bandura 自我效能理论、心流研究 | 完成一件有挑战性的新事物，自我效能感显著提升 |
| 财务 | Gilovich 体验性消费研究 | 花钱买体验比花钱买物品，长期幸福感贡献更高 |

基于这些研究，每类行动原子有初始影响区间（不是精确值）：
- 探索新地点 → 环境 +10~20，可能性 +10~15
- 陌生社交互动 → 社交 +15~25（研究支撑度：非常高，可重复）
- 创造类活动 → 事业 +10~20，可能性 +5~10
- 独处反思活动 → 健康 +8~15（个体差异较大）
- 户外身体活动 → 健康 +15~30（最扎实的研究领域之一）

这些是先验值，随用户数据积累逐步被实测均值替换。

---

#### 来源 2：用户生成数据（运营后逐步接管）

用户每次打卡的情绪贴纸 + 文字，构成真实经验数据库：

- N > 30 完成后：「70% 的人做完选了「治愈」」—— 这是有统计意义的陈述
- N > 100 后：可以呈现情绪分布直方图，展示真实的体验多样性
- 纵向数据（更难但更有价值）：做了这张卡的用户，30 天后六维评分是否有显著变化？

数据替换时机：
- 冷启动（N < 30）：纯编辑先验 + 研究区间
- 成长期（30 < N < 100）：数据辅助编辑内容，开始插入真实比例数据
- 成熟期（N > 100）：数据主导，编辑做质量审核
- AI 合成阶段：从用户打卡文字中归纳体验模式，标注「综合了 N 个人的感受」

---

#### 来源 3：行为科学写作方法（预演文字怎么写才有用）

这是最容易被忽视但影响最大的一层：**预演的写法本身，会影响用户最终是否去做。**

**实施意图理论（Gollwitzer）**

「If X, then Y」式的提前规划，能将行为跟随率提升 2-3 倍（大量实验研究支持）。预演不只是描述会发生什么，还要嵌入「如果遇到困难，就这样」的路径。

> 错误写法：「你可能会感到有点尴尬。」
>
> 正确写法（实施意图）：「如果你到了觉得很尴尬，那是正常的——这种感觉几乎每个人都有，通常在 15 分钟后消失。你只需要决定待到那个时间点。」

后者不只是描述，它是一个「预装的应对策略」，显著提升用户跨过困难的概率。

**峰终定律（Kahneman）**

人对体验的记忆由峰值和结尾决定，而非全程平均。预演应该明确标出：

- 哪个时刻通常是最有感觉的（「通常在 30 分钟后开始变得不一样」）
- 结束的时候大多数人什么感觉（「离开时大多数人说感觉比进来时轻」）

而不是均匀描述全程。

**情感预测校正（Wilson & Gilbert）**

人系统性地高估未来体验的强度和持续时间——无论好坏。预演可以温和校正这个偏差：

- 「这比你可能预期的更快就结束了，哪怕中间有些不舒服」
- 「做完以后，几乎没有人后悔去做了」（这是有实证基础的陈述）

**维度影响分的展示原则**

不展示假精确。

> ❌「可能性 +18.3」（精确数字反而破坏信任）
>
> ✅「可能性 ████░░ 预期 +10 到 +20」（区间表达更诚实）

如果数据不足，宁愿不展示维度变化，也不要展示无依据的数字。参考天气预报的「降雨概率」写法——概率比「会/不会下雨」更诚实，也更被用户信任。

---

### 预演文字的标准时间结构

形成统一的文字模板，降低编写成本，也让用户形成预期：

```
开始时：[你大概会经历什么，包括可能的不适]
转折点：[通常在哪个时刻发生改变]
结束时：[大多数人离开时的感受]
如果遇到 [具体障碍]：[应对方式]（实施意图植入）
```

示例：
> 到了之后头 10-15 分钟可能有点不知道该做什么——这是正常的，因为这个地方对你是新的。
> 通常在你开始走动之后，会找到一个节奏。
> 离开的时候，大多数人说感觉比进来时轻一些，哪怕什么「大事」都没发生。
> 如果你到了之后感觉「这有什么意思」，给自己再 15 分钟——这种感觉几乎每次都会在那之后转变。

---

### 预演的演进路径

预演不是一次性写好的内容，而是随产品数据密度增长的活数据：

| 阶段 | 预演来源 | 用户看到的标注 |
|------|---------|--------------|
| 冷启动 | 编辑撰写 + 研究区间 | 不标注（无需解释来源） |
| 成长期 | 编辑基础 + 真实情绪数据 | 「已有 [N] 人做过这件事」 |
| 成熟期 | AI 合成用户文字 + 编辑审核 | 「综合了 [N] 个人的感受」 |
| 长期 | 完全来自用户数据，实时更新 | 数据时效性标注 |

这条演进路径需要在产品里有一个细节体现：让用户知道「这个预演会越来越准，因为有越来越多的人做了这件事」。这个认知能增加用户对预演的信任，也能增加他们打卡的动力（「我的记录会帮到下一个人的判断」）。

---

### Phase 0 预演写作指南：在没有任何数据的情况下如何保持可信

这是当下最实际的问题：卡片库 v2.0 里的预演全部是编辑写的，没有任何真实用户体验支撑。这不是问题本身，但需要用正确的写作策略来处理，否则预演会在用户第一次「对不上」之后永久失去可信度。

**核心写作原则：经验感 > 准确性**

用户无法判断预演准不准（他们还没做过），但能感觉出来这段文字是不是真实做过的人写的。经验感来自三个信号：

1. **具体的时间点**：「头十分钟」「第三或第四首歌」「大约 30 分钟后」比「刚开始」「一段时间后」感觉真实十倍
2. **非线性的体验描述**：真实体验不是直线上升的。「可能会拿到很好的东西，也可能就那样」比「你一定会有所收获」更可信
3. **至少一个反直觉的细节**：用户做之前不会想到但听了觉得「有道理」的东西——这是最强的可信度信号

示例对比：
> ❌ 「一个人去探索城市是非常有意义的体验，你会看见不同的风景，拓展你的视野。」
>
> ✅ 「终点站通常是你不熟悉的地方。出了地铁，你完全不知道周围是什么，没有任何计划，只有你和那个地方。这种没有目的地的出发感，比任何精心安排的行程都让人记得住。」

后者提供了一个具体的认知框架（终点站 = 不熟悉的地方），一个具体的感受时刻（出了地铁），和一个反直觉的结论（比精心安排的更难忘）。

**语气禁忌：不能用的词**

这些词会立刻打破信任，因为它们是在替用户做判断，而不是描述事情本身：

| 禁用 | 改用 |
|------|------|
| 你一定会 / 你肯定会 | 通常 / 很多人说 / 大概率 |
| 非常有意义 | 直接描述发生了什么 |
| 改变你的想法 | 留下具体的什么 |
| 让你感觉更好 | 大多数人说做完... |
| 一定不后悔 | 几乎没有人后悔去做了（如果是真的） |

**处理不确定性：诚实是最好的可信度策略**

冷启动预演不应该假装什么都知道。对于结果不确定的卡片，直接说出来反而建立信任：

> 「可能会拿到很好的东西，也可能就那样。但有意思的是，结果好不好反而不是这件事的重点——」
>
> 「不一定每次都会有什么大感悟。但做过之后，那条路在你的地图里不再是空白。」

这种诚实不会打消用户行动的欲望——它会建立「这个 App 不是在忽悠我」的基础信任，这比任何数字背书都更持久。

**维度影响分的诚实展示**

Phase 0 的维度数字（`health: {min: 5, max: 12}`）全部是编辑估算，没有实证基础。这不是问题，但展示方式要与这个现实匹配：

- Phase 0：只展示方向（哪些维度会受影响），不展示具体数字
- Phase 1（有真实打卡数据后）：展示区间，标注数据量（「基于 N 次打卡的估算」）
- Phase 2+：数据主导，区间收窄，可信度自然增加

> ✅ 显示：「健康 ↑」「环境 ↑」（方向感）
>
> ❌ 显示：「健康 +12」（无依据的精确）

这个原则贯穿整个产品生命周期：**宁愿展示少一点，不要展示不可信的东西。**
- [ ] 卡片池的初始内容来源和分类体系
- [ ] 首页核心交互形式的确定
- [ ] 六维模型的量化方式
- [ ] 热力地图的产品细节
- [ ] 留存机制的深入设计
- [ ] 社区共建的规则和激励
- [ ] 产品名？调性？面向什么人群？
- [ ] 移动端优先还是网页端？
- [ ] 商业模式初步想法？
