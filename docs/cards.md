# 卡片系统深度设计

> 专项脑暴文档。包含调研提炼、对已有思路的延伸、以及我的独立判断（会标注「我的判断」）。

---

## 一、核心命题：卡片是什么？

卡片不是内容对象，是**函数**。

```
Card = f(触发原子, 行动原子, 情境原子, 结果原子)
```

这个视角的根本意义在于：卡片不是被"写出来"的，而是被"组合出来"的。原子有限，组合无限。

---

## 二、原子体系（Atomic System）

这是整个卡片系统的底层。先定义原子，再谈组合。

### 2.1 触发原子 —— 什么时候这张卡片是合适的？

**时间类**
- 工作日晚上（19:00-23:00，有限时间）
- 休息日下午（大块自由时间）
- 小长假（3 天以内）
- 大假期（5 天以上，可出行）
- 清晨（6:00-9:00）
- 深夜（22:00 后）

**财务类**
- 本月紧张（预算极低）
- 正常状态（日常消费能力）
- 有闲钱（可以花点不一样的）
- 说走就走（大额旅行预算）

**情绪/心理状态类**
- 疲惫/需要恢复
- 无聊/空洞
- 焦虑/压力大
- 心情好/想庆祝
- 孤独感
- 好奇心高涨
- 想改变点什么

**特殊事件类**
- 生日前后
- 刚换工作/毕业/搬家
- 刚结束一段关系
- 即将到来的节气/节日
- 刚完成一件大事

**季节/天气类**
- 晴天
- 下雨天
- 第一场雪
- 春天特有（樱花 / 新绿）
- 夏夜
- 秋天

---

### 2.2 行动原子 —— 这张卡片的核心动作是什么？

| 行动类 | 例子 |
|--------|------|
| 探索 [地点类型] | 探索陌生街区、第一次去某类场所 |
| 尝试 [食物/饮品/产品] | 第一次吃某种东西、盲买一张唱片 |
| 创造 [产出物] | 写一封信、做一道菜、拍一卷胶卷 |
| 学习 [技能/知识] | 学 10 个新单词、了解一个从没想过的话题 |
| 连接 [对象] | 给某人打电话、和陌生人聊一句 |
| 观察 [现象] | 看日落、看人流、在咖啡馆发呆 |
| 移动 [方式] | 用走路代替打车、爬山、游泳 |
| 反思 [主题] | 写今天三件好事、给自己写一封信 |
| 收集 [物件/记忆] | 捡一颗好看的石头、拍一张只给自己看的照片 |

---

### 2.3 情境原子 —— 如何做，和谁做？

**社交维度**
- 独自完成
- 和固定的某人（伴侣 / 老朋友 / 父母 / 孩子）
- 小团体（3-5 人）
- 和陌生人

**地点维度**
- 家里 / 不出门
- 就在附近（2 公里内）
- 城市内探索
- 跨城出行
- 自然环境

**计划维度**
- 完全即兴 / 说做就做
- 简单准备（当天能完成）
- 需要提前计划（提前几天）
- 需要较长准备（提前几周）

**记录维度**
- 不需要记录任何东西
- 拍一张照片就够
- 写几句话
- 完整记录体验

---

### 2.4 结果原子 —— 这张卡片主要影响哪个维度？

对应六维，每张卡片有主要方向和次要方向（权重不同）。

```
健康↑ / 财务↑ / 事业↑ / 社交↑ / 环境↑ / 可能性↑
```

一张卡片通常有 1 个主要维度 + 1-2 个次要维度。

例：「一个人去从没去过的街区走两小时，不带目的地」
- 主：环境↑（物理世界探索）
- 次：可能性↑（打开新区域）+ 健康↑（步行）

---

### 2.5 原子组合的意义

**理论组合量**
- 触发原子约 20 个维度值
- 行动原子约 9 类 × N 具体值
- 情境原子约 12 个维度值
- 结果原子 6 个维度

组合空间是指数级的，远远大于任何人工能穷举的卡片数量。

**实际价值**
1. **生成**：可以用原子组合系统性地生成候选卡片，而不是靠灵感
2. **验证**：对约束规则（如"独自 + 陌生人"不能共存）自动过滤
3. **个性化**：用户的历史数据等于"原子偏好向量"，推荐时偏向这个向量
4. **寻找空白**：枚举哪些合理的原子组合当前卡库里没有覆盖
5. **邻域推荐**：每次只改变一个原子，生成"比上次稍微更难一点"的卡片

## 三、卡片分类体系

在原子的基础上，做两层分类：**规模层级**和**内容域**。

### 3.1 规模层级（Scale Tiers）

| 层级 | 时间投入 | 准备成本 | 典型例子 |
|------|----------|----------|----------|
| 微粒卡 | < 15 分钟 | 零 | 今天上床不带手机 |
| 日常卡 | 1-4 小时 | 低 | 去一家没去过的餐厅 |
| 事件卡 | 半天-整天 | 中 | 周末骑行 40 公里 |
| 大爆炸卡 | 多天 | 高 | 「不要错过这个春天」—— 含 5 张子卡 |
| 里程碑卡 | 长期 | 极高 | 高考 / 毕业 / 考研 |

**我的判断**：微粒卡是被严重低估的类型。它的价值不在于本身，而在于它是**建立用户行动习惯的最小单位**。第一周只推微粒卡，用户建立起"Vibe 推荐的事我真的去做了"的信任感，后面才能推动更大规模的体验。

---

### 3.2 内容域（Content Domains）

| 域 | 关键词 |
|----|--------|
| 食物 & 饮品 | 探索味觉边界 |
| 艺术 & 文化 | 展览、演出、书影音 |
| 自然 & 户外 | 身体 + 物理世界 |
| 学习 & 技能 | 长期成长 |
| 社交 & 人 | 关系维度 |
| 身体 & 运动 | 不限于健身 |
| 内省 & 写作 | 向内看 |
| 创造 & 手作 | 产出导向 |
| 冒险 & 打破 | 舒适区挑战 |
| 家 & 日常 | 改变日常的小方式 |

---

### 3.3 特殊卡型

**野卡（Wild Card）**
- 和用户历史偏好刻意相反，由系统强制注入
- 每天 1-2 张，在多样性注入步骤里强制插入
- 目的：打破算法气泡，制造发现感

**野卡视觉设计**

野卡有两个视觉层：

① 卡片本体：右上角一个小标记，用渐变色圆角方块（不用 emoji），内部是一个骰子点阵图形（⚄ 的几何化版本），色系用紫→橙渐变，与普通卡片的白/浅色背景形成识别度。标记旁边只有两个字：**意外**。

② 进入卡片栈前的视觉提示：野卡在卡片栈里的缩略位置显示时，背景有轻微的光晕扩散动画（静止状态下的 1-2% 动态），提示「这张和别的不太一样」。

**主动切野卡入口**

在主卡片界面底部，常驻一个骰子形状的浮动小按钮，文案：「换一张意外的」。

- 点击后当前卡片滑出，从野卡池里随机抽一张从右侧飞入
- 按钮每天可用 3 次（避免用户用它替代正常浏览）
- 用完后变灰，提示「今天的意外已用完，明天再来」
- 触发动画：骰子图标点击后做一次快速旋转，卡片飞入时有轻微弹性效果

这个设计的核心是：野卡不只是系统行为，用户可以主动召唤，给了用户一种「我想要惊喜」的自主感。

**盲盒卡**
- 不显示内容，只显示：预估时间 + 在你家 X 公里内
- 右滑接受后才揭晓
- 适合"我想有惊喜"状态下
- 参考：Randonautica

**地点绑定卡**
- 到达特定地点才触发/解锁
- 某咖啡馆、某公园、某书店
- 地点到达是一种隐形的执行力过滤器

**挑战卡**
- 有截止时间
- 如"本周之内，找到一家你觉得有故事的老店"
- 紧迫感增加行动率

**社区卡**
- 来自其他用户的真实体验，提炼成标准卡片格式
- 需要通过质量过滤才能进入流通

## 四、置信度（Confidence）

置信度回答：**这张卡片推给这个用户，有多大把握是对的？**

### 4.1 置信度来源层级

| 层级 | 来源 | 权重 |
|------|------|------|
| L5 个人行为 | 这个用户自己做过类似的事并有正向反馈 | 最高 |
| L4 行为聚类 | 相似画像用户的高评价 | 高 |
| L3 社会热度 | 大量用户做过且评价好 | 中 |
| L2 编辑判断 | 人工标注的高质量卡片 | 中 |
| L1 探索注入 | 新卡片，数据极少，作为野卡插入 | 低（但故意如此） |

### 4.2 置信度对展示的影响

用户不直接看到置信度数字，但它影响：
- **出现时机**：高置信度卡片在用户状态匹配时优先出现
- **推荐语言**：
  - L5：「你之前喜欢 X，这个和它有点像」
  - L4：「和你相似的人评价很高」
  - L3：「已有 2847 人做过」（无个人化关联）
  - L2：只显示编辑精选标记
  - L1：不解释来源，直接出现（野卡标记）

### 4.3 「我的判断」：置信度不等于准确率

高置信度只代表推荐系统认为你会喜欢。但最有价值的体验往往是你**事先不知道自己会喜欢**的。

所以置信度高 → 增加转化率，置信度低（野卡）→ 增加发现感。两者都需要，不能只优化前者。

---

## 五、可执行度（Executability）

可执行度回答：**这个用户现在能不能做这件事？**

不同于置信度（用户是否想要），可执行度是**当前状态与卡片需求的匹配度**。

### 5.1 可执行度 = 用户当前状态 × 卡片需求

**用户当前状态输入**
- 时间（当前是工作日夜晚 vs 假期白天）
- 财务（本月状态）
- 地点（在家 / 在公司 / 在路上 / 在外地）
- 同行人（独自 / 有伴侣 / 有孩子）
- 体力（可从近期行为推断，或用户偶尔自报）
- 季节/天气

**卡片需求**
- 最少时间
- 预算范围
- 所需地点类型
- 最适合的同行人
- 季节限制

### 5.2 可执行度分级

| 级别 | 含义 | 展示方式 |
|------|------|----------|
| 绿色 | 现在所有条件都满足，可以立刻做 | 正常展示，标记"今天就可以" |
| 黄色 | 1-2 个条件不完全符合，但可以调整 | 展示，说明"这个周末更合适" |
| 橙色 | 需要提前规划，不适合冲动执行 | 展示，带"存入计划"引导 |
| 灰色 | 核心条件完全不符合（预算、季节等） | 不在今日主流中出现，但可搜索/存档 |

## 九、等待盒子（Waiting Box）

等待盒子是专门承接「现在不做、但不想放弃」的卡片的空间。它是整个卡片系统里节奏最慢的一层，与主流的快速滑动形成对比。

### 9.1 等待的四种类型

卡片进入等待盒子有四条路径，每种类型有不同的行为逻辑：

| 类型 | 触发方式 | 重新浮出逻辑 | 视觉状态 |
|------|----------|--------------|----------|
| **等季节** | 系统判断季节不符 / 用户滑走时选「等时机」 | 季节到来时系统自动推送通知 + 回到主流 | 卡片覆盖一层半透明的季节图标（落叶/雪花/花瓣），像盖着一层薄雾 |
| **等条件** | 系统判断财务/地点不符 / 用户选「等时机」 | 对应条件信号变化时（如新的财务状态）自动浮出 | 卡片底部有一行灰色小字「等你 XX 的时候」 |
| **等心情** | 用户明确说「现在状态不对」 | 弱：依赖状态信号。用户也可手动提前激活 | 卡片有轻微的「睡眠」呼吸动画（透明度 0.8-1.0 循环） |
| **我的收藏** | 用户主动存入，无条件绑定 | 不自动浮出，完全由用户手动管理 | 正常卡片样式，加一个书签角标 |

### 9.2 等待盒子入口与界面

**入口**：主界面右上角或底部 tab，一个小盒子图标，旁边有未读数字标注（「2 张卡片等你」）。

**界面结构**：
```
┌─────────────────────────────────┐
│  等待盒子                  [管理] │
│                                 │
│  🌸 等季节 (3)                  │
│  ┌──────┐ ┌──────┐ ┌──────┐   │
│  │赏樱花│ │暑夜夜│ │看雪 │   │
│  └──────┘ └──────┘ └──────┘   │
│                                 │
│  ⏳ 等条件 (2)                  │
│  ┌──────┐ ┌──────┐             │
│  │穷游成│ │...  │             │
│  └──────┘ └──────┘             │
│                                 │
│  💤 等心情 (1)                  │
│  ┌──────┐                      │
│  │陌生人│                      │
│  └──────┘                      │
│                                 │
│  🔖 我的收藏 (7)                │
│  ──────────────────────────    │
└─────────────────────────────────┘
```

### 9.3 等待盒子内的卡片交互

- **右滑**：「我现在准备好了」→ 立刻提升到主队列，状态从等待变为已收藏（活跃）
- **左滑**：「算了，不等了」→ 移出等待盒子，判为长期不感兴趣
- **点击展开**：查看这张卡片的详情 + 系统说明「等什么」+ 预计什么时候条件会满足
- **无限期保留**：不设过期时间，等待盒子是安静的，不催促

### 9.4 系统主动唤醒

条件满足时，系统发送通知：

> 「你在 3 周前存了一张赏樱的卡片，北京的樱花本周开了。」

通知点击后直接进入该卡片的详情页，一键「就现在做」。

唤醒通知的设计原则：**有时效性的才推，普通的「条件满足了」不推通知，只在等待盒子里更新状态标记。** 季节是最典型的推通知场景，因为窗口期短。

红线分两类：系统红线（硬性）和用户红线（软性）。

### 6.1 系统红线（硬性，绝不出现）

| 类型 | 例子 |
|------|------|
| 安全红线 | 需要专业训练才能安全执行的极限运动，在卡片里轻描淡写推出去 |
| 法律红线 | 任何违法行为，包括轻微的灰色地带 |
| 伦理红线 | 任何可能伤害他人或剥削性质的内容 |
| 平台基调红线 | 制造羞耻感、自我否定的内容；助长成瘾行为的内容 |

补充说明：系统红线不只是内容层面，**推荐时机也有红线**。例如：深夜 11 点不推"开车去外地"；检测到用户近期高度焦虑状态时，不推"参加陌生人局"。

### 6.2 用户红线（软性，默认关闭但可解锁）

| 类型 | 默认状态 | 解锁方式 |
|------|----------|----------|
| 独处类挑战 | 开 | 不需要解锁 |
| 陌生人社交 | 关 | 用户主动打开「社交挑战模式」 |
| 高强度身体挑战 | 关 | 用户打开「运动激进模式」 |
| 高消费体验 | 关 | 用户设定预算上限解锁 |
| 特定内容类型（如恐怖片） | 关 | 用户打开「打破舒适区模式」 |

**我的判断**：用户红线的设计原则是「默认保守，解锁有仪式感」。不要在 Onboarding 时让用户一次性选择所有偏好。而是在具体卡片旁边放一个「想要更多这类挑战？」的小入口，让解锁行为和具体场景绑定，用户更容易做决定。

## 七、每日卡片数量策略

**不是固定的，是动态收敛的。**

冷启动期的核心矛盾：算法还不了解你，推荐精准度低，固定 12 张意味着大概率都是噪音。应该反过来想——**越不了解你，越需要给你更多机会找到对的那张。**

| 阶段 | 触发条件 | 每日数量 | 逻辑 |
|------|----------|----------|------|
| 探索期 | 互动总次数 < 50 | 不限量，自由滑 | 讨好用户，快速建立偏好画像 |
| 收敛期 | 完成第 1 张打卡后 | 20-25 张 | 开始引入轻微稀缺感 |
| 标准期 | 使用满 14 天 + 完成 3 张打卡 | 15 张 | 卡片质量开始上升 |
| 精准期 | 使用满 30 天 + 完成 5 张打卡 | 12 张 | 少而精，每张都是为你选的 |

这个动态本身可以成为一个叙事：「你用的越久，每天的卡片越少，但越准确。」这是产品价值的可见化，而不是减少供给。

## 八、层层筛选管道（Filtering Pipeline）

```
全量卡片池
    │
    ▼
① 硬红线过滤
  └→ 移除安全 / 法律 / 伦理违规卡片（永久）
    │
    ▼
② 用户红线过滤
  └→ 移除用户关闭的内容类型
    │
    ▼
③ 季节 / 时效过滤
  └→ 移除季节不符、已过期的内容
    │
    ▼
④ 可执行度标注
  └→ 对每张卡标注绿/黄/橙/灰，灰色进等待盒子，不进当日候选
    │
    ▼
⑤ 重复过滤
  └→ 移除与最近 7 天已展示/已完成卡片过于相似的内容
    │
    ▼
⑥ 置信度排序
  └→ 按个性化得分对候选卡片排序
    │
    ▼
⑦ 多样性注入
  └→ 强制保证每日卡片在「内容域」「规模层级」「社交维度」的多样性
    │
    ▼
⑧ 野卡注入
  └→ 强制插入 1-2 张打破偏好的卡片（见野卡设计）
    │
    ▼
⑨ 每日精选
  └→ 独立逻辑，不从 ⑧ 的池子里选，单独置顶展示
    │
    ▼
最终每日卡片栈（N 张 + 1 精选）
```

**步骤 ⑦ 多样性注入的硬性约束**：
- 同一内容域最多出现 3 张
- 至少 1 张微粒卡（保持行动惯性）
- 至少各含 1 张独处卡 / 社交卡
- 规模层级分布：微粒≥1，日常≥3，事件≥2，大爆炸/里程碑≤2

## 八、卡片状态机（Card State Machine）

每张卡片对每个用户都有独立的状态。

```
未见过
   │
   ▼
已展示
   ├──左滑──→ 拒绝（长期不再出现，但不删除）
   ├──上滑──→ 已做过（沉淀为历史记录，不再推荐）
   └──右滑──→ 感兴趣
                │
                ▼
             已收藏（队列中）
                │
                ├──时间流逝──→ 褪色（温和提醒）
                │                │
                │                └──继续不动──→ 淡出（但不消失）
                │
                └──打卡完成──→ 已完成
                                  │
                                  ▼
                               永久档案
                               （发光、不可删除、地图点亮）
```

**褪色机制**：右滑但超过 7 天未行动，卡片在收藏列表里轻微去饱和。不是惩罚，是「它还在等你」的视觉表达。

**「我的判断」：左滑信号太粗糙**

「左滑」包含至少三种完全不同的意思：
1. 我做过了（实际上应该上滑，但用户不知道）
2. 这类事我不喜欢（长期信号）
3. 现在不合适，但以后可能（短期信号）

建议：左滑后 1.5 秒出现一个极简 prompt，只有两个选项（不要更多）：
- 「不太感兴趣」
- 「现在不合适」

捕获这个区分，对推荐质量的提升非常大，而额外摩擦极小。

## 十一、选择预演（Pre-Experience Preview）

**每张卡片都有预演。** 但预演的形态因卡片类型不同而分叉。

### 11.1 单卡预演（适用于微粒卡 / 日常卡 / 事件卡）

在用户决定「去做」之前，展示三层内容：

**① 过程预演**：中间大概会经历什么

> 「你可能需要提前查一下地址。到了之后，头 10 分钟可能有点尴尬，不知道在哪里站。通常 20 分钟后就自然了。结束后大概率会在路上多走一会儿。」

**② 结果预演**：可以期待什么（带概率感）

> 「不是每次都很爽。做过这个的人里，大约 70% 说'会再去'，20% 说'做一次就够了'，10% 说'没什么感觉'。」

**③ 维度预演**：六维的预期变化

```
健康    ████░░  +15
社交    ██░░░░  +8
可能性  █████░  +20
```

**设计原则**：
- 不要过度渲染，包含不确定性
- 极简，三屏以内完成，不要变成攻略
- 前期人工编写，后期 AI 生成

---

### 11.2 大爆炸卡预演（含多张子卡的复合体验）

大爆炸卡不直接进入三层预演，而是先展示「旅程概览」。

**旅程概览页**：

```
┌─────────────────────────────────┐
│  不要错过这个春天                 │
│  5 个体验 · 建议 3 周内完成       │
│                                 │
│  ① 去最近的公园坐一小时，不带手机  │  ← 可见
│  ② 买一束鲜花放在家里            │  ← 可见
│  ③ 找一个没去过的城市公园         │  ← 可见
│  ④ ████████████████             │  ← 锁定（开始后解锁）
│  ⑤ ████████████████             │  ← 锁定（开始后解锁）
│                                 │
│  整体预期：                      │
│  可能性 +45 · 健康 +20 · 环境 +30│
│                                 │
│  [先看子卡详情]   [接受这组体验]  │
└─────────────────────────────────┘
```

**关键设计决策**：

1. **子卡渐进解锁**：前 2-3 张子卡可见，后续子卡在开始后随完成进度解锁。理由：像连续剧，制造「下一集是什么」的期待感。如果一开始全部展示，用户可能因为某一张子卡不喜欢而放弃整组。

2. **接受大爆炸卡 = 进入「任务模式」**：主界面出现一个置顶的进度条，显示「春天探索 2/5」。子卡按顺序出现在主流中，每完成一张才解锁下一张的预演。

3. **整体维度预演 vs 子卡维度预演**：旅程概览只展示整体维度变化。进入子卡详情时，每张子卡有自己的单卡预演。

4. **可以中途退出**：大爆炸卡没有强制完成。未完成的子卡回到等待盒子的「我的收藏」里。

## 十二、完成后的体验（Post-Experience）

完成打卡是整个闭环里情绪峰值最高的时刻。

### 12.1 打卡流程

```
打卡按钮
    │
    ▼
(可选) 拍照或跳过
    │
    ▼
选情绪贴纸（必选，6选1）
    │
    ▼
(可选) 一句话
    │
    ▼
选择可见范围
    │
    ▼
完成动画
```

**情绪贴纸（6 个，不多不少）**：

治愈 · 开心 · 释然 · 疲惫 · 突破 · 平淡

「平淡」这个选项非常重要。如果只有正向情绪，用户会觉得「这次没什么感觉」就无法打卡，形成沉默螺旋。平淡是诚实的，而且用户感受到被接纳。

---

### 12.2 可见范围设置

打卡完成前，最后一步是选择可见范围。**默认私密**，降低新用户焦虑。

```
这次体验，让谁看见？

  ○  🔒  仅自己       ← 默认选中
  ○  🌍  所有人
```

（故意只给两个选项，不做「朋友可见」中间档——社交图谱尚未建立，中间档没有意义。后期有朋友关系后再加。）

**公开打卡会贡献**：
- 你的一句话（匿名处理后）可能出现在「情境化社会认同」里被他人看到
- 你的位置数据贡献地图热力
- 你的情绪标签计入卡片的整体情绪分布（「70% 人感到治愈」）
- 卡片的全局完成计数 +1（影响碎片卡编号）

**私密打卡仍然贡献**：
- 你的个人地图点亮
- 你的六维数据更新
- 你的碎片卡档案（数字来自全局计数，私密打卡也计入）

**设计取舍**：私密打卡是否应该计入全局完成数字（影响别人看到的编号数字）？我倾向于计入，因为「你做了」本身就是事实，只是不展示身份。但这需要在隐私政策里说清楚。

---

### 12.3 完成动画

1. 情绪贴纸选定后，卡片开始发光变色
2. 「提交」后：卡片以 flip 动画翻转，背面出现碎片卡（编号 + 情绪词 + 日期）
3. 碎片卡从屏幕飞向右下角的档案图标，消失
4. 同时：一个光点在地图上落下，扩散动画（如果公开：热力图更新）
5. 手机轻震

整体时长控制在 2 秒以内，不能拖太长。

## 十三、其他补充机制

### 13.1 「几乎做了的卡片」（Near-Miss Card）

用户右滑表示感兴趣，5 天后还没行动。

不是直接褪色，而是主动变形：

> 「上次你对这个感兴趣，但没去。这里有一个更小的版本，5 分钟就能开始：[降级版本卡片]」

原理：承诺一致性 + 降低门槛。用户右滑的那个「小承诺」，用降级版本来激活。

### 13.2 情境化社会认同

现有思路里的社会认同（「2847 人做过」）是聚合数据，说服力中等。

可以做到更精准：

> 「一个同样在北京、独自体验的人说：比想象中难，但比想象中值。」

不显示是谁，只显示与用户相关的属性（同城市、同社交模式）。这种「我和他像」的感觉比数字更有说服力。

实现上不复杂：给用户的每条打卡记录打上原子标签，展示时做属性匹配。

### 13.3 卡片「衰减参数」

不是所有卡片都永远有效。有些有自然的时效性：

| 类型 | 衰减行为 |
|------|----------|
| 季节限定（春日赏花） | 过季自动退场，明年再出现 |
| 时效紧迫（这家店即将关门） | 倒计时显示，期限后消失 |
| 即将发生的活动（演出/展览） | 固定推送时间窗口 |
| 心理状态限定（想法多的时候看看这个） | 当状态信号触发时才出现 |
| 常青内容 | 永远有效，用户长期可见 |

### 13.4 拒绝信号的精细化

这点在第八节提到过，值得单独强调一次。

左滑的额外 prompt（选一个）：
- 「不太感兴趣」→ 降低这类原子的推荐权重
- 「现在不合适」→ 保留卡片，等时机匹配再推

这两个信号在推荐系统里的处理完全不同，但对用户来说只是轻点一下。

## 十四、冷启动的卡片策略

在没有用户数据、没有大量 AI token 的情况下，如何建立第一批卡片？

### 阶段 A：人工枚举（第一个月）

用原子体系枚举最重要的 300 张卡片：

1. 列出最常见的 20 种「触发情境」（周末空闲 / 疲惫想恢复 / 清明假期……）
2. 每种情境配 3-5 个内容域
3. 每个内容域下写 2-3 张具体卡片
4. 确保覆盖：所有规模层级、独处和社交两个向度、低中高三个消费层

目标：300 张精品 > 3000 张凑数。

### 阶段 B：AI 离线批量生成（第二个月）

- 输入：原子体系 + 已有的 300 张卡片作为范本
- 输出：候选卡片大批量生成
- 人工过滤：只需审核（是/否），不需要创作
- 目标快速扩充到 2000+ 张

### 阶段 C：AI 实时生成（后期）

- 对特定用户的特定情境，实时生成个性化卡片
- 有了充分的用户数据后才开启，避免生成质量不可控
