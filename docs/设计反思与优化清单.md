## 克制标签说明

| 标签 | 含义 |
|------|------|
| `[ 保留 ]` | 符合克制原则，推进 |
| `[ 克制版 ]` | 方向对，需做减法 |
| `[ 存疑 ]` | 有真实价值，但与调性存在张力，待讨论 |
| `[ 暂缓 ]` | 留档，不排期 |

核心判断标准：**帮用户做他们本来就想做的事（内驱），而不是让用户为产品服务（外驱）。**

## 一、产品层面

### P1. 右滑到打卡的转化 🔴

**问题**：Pinterest 困境——右滑无成本，做到才有价值。用户可能积累一个永远不执行的队列。

**建议**：

- **实施时间锚定** `[ 保留 ]`：队列卡片可绑定"这个周六下午"。行为科学里的「执行意图」——帮用户把模糊意愿转化为具体计划，没有压力，只是帮助。

- **「就地可做」卡片比例** `[ 保留 ]`：大量卡片要求已出门才能做。在卡片设计原则中明确比例（建议 ≥30% 为室内/随时可做），这是卡片质量问题，不是功能问题。

- **承诺装置** `[ 存疑 ]`：「告诉一个朋友」利用社会压力提升执行意愿，有效但略带操纵感。替代方向：「给自己发一条备忘」——同样利用承诺心理，但不消耗社交资本。**待决策：用哪种，或者都不做？**

**现有对应设计**：首页架构文档有队列设计，未处理执行转化。❌ 缺失。

---

### P2. 传播机制 🔴

**整体重新框架**：不追求「病毒传播」，追求「值得分享的瞬间」——用户主动想让朋友看到的东西，而不是被机制征召。

**各子项决策**：

| 子项 | 决策 | 说明 |
|------|------|------|
| 城市探索进度分享（「我点亮了 X%」） | `[ 克制版 ]` ✅ 做 | 分享自己的真实探索记录，内驱。去掉横向排名比较，只展示自己进度 |
| 挑战/点名机制 | `[ 暂缓 ]` | 社会压力驱动，制造义务感，与产品调性冲突 |
| 先行者/早鸟碎片 | `[ 暂缓 ]` | 内部逻辑可以保留（记录完成序号），但不外显、不制造稀缺感，择机再议 |
| 季节性内容 | `[ 克制版 ]` ✅ 做内容，暂缓运营 | 季节性卡片（立春/梅雨/秋日）随自然节律上线，有温度。但不做季节性运营 campaign，不做倒计时，不做「限时消失」 |
| 季末 Wrapped 报告 | `[ 暂缓 ]` | 随季节性运营一起暂缓，未来时机成熟一并做 |

**现有对应设计**：碎片收藏体系文档有分享卡，但无主动传播机制设计。❌ 缺失。

---

### P3. Onboarding 转化问题 🔴 `[ 保留 ]`

**问题**：产品价值（做了才体验到）天然延迟，前 3 次交互就可能失去用户。现有设计是直接进首页做味道采样。

**建议**：逆向 Onboarding——在用户刷了 5 张卡后插入一个时刻：

> 「告诉我一件你最近做了的、让你感觉还不错的小事」

AI 归类后立刻返回一张相关卡片。用户先体验，再被理解。没有问卷，只有倾听。

**现有对应设计**：用户旅程文档有 Onboarding 章节，策略是主动问卷式。⚠️ 需重新讨论。

---

### P4. 粘性机制缺口 🟡

| 问题 | 建议 | 标签 |
|------|------|------|
| 缺「生活日历」感 | 引入节气/时令内容，跟着自然节律走 | `[ 保留 ]` |
| 缺叙事连续性 | 引入「主题包」（城市探索者 Vol.1），让用户感觉在完成一段旅程 | `[ 保留 ]` |
| 缺「同行」机制 | 你和朋友各自完成同一张卡，可以互看碎片——不是主动社交，是自然发现的惊喜连接 | `[ 保留 ]` |
| 缺分时体验 | 不做显式时间感知切换；做功能差异化：晚间默认打开回顾/记录入口，早晨默认首页即可 | `[ 克制版 ]` |

**现有对应设计**：用户旅程文档部分提及，未形成设计规范。⚠️ 部分缺失。

---

### P5. 六维框架潜在问题 🟡

| 问题 | 建议 | 标签 |
|------|------|------|
| 「可能性」维度定义模糊 | 迭代中明确，与其他维度划定边界 | `[ 保留 ]` |
| 「财务」维度难以推断 | 允许该维度暂时空着，显示「数据不足」比显示不准确的分数更诚实 | `[ 存疑 ]` |
| 分数可被右滑刷分 | 只有打卡完成才能大幅影响维度分，右滑仅做轻微预测权重 | `[ 保留 ]` |
| 缺乏自然衰减 | 30 天半衰期向均值漂移——分数是生活当下的映射，不是永久成就 | `[ 保留 ]` |
| 绝对值展示意义有限 | 改为增量展示（Δ 值 + 趋势）：「本月健康 ↑12」比「健康: 75」更有意义 | `[ 保留 ]` |

**现有对应设计**：六维框架文档有评分设计，无防刷和衰减。❌ 缺失。

### P7. 地理覆盖局限性 🟡 `[ 保留 ]`

**问题**：大量卡片依赖城市密度（博物馆、独立书店、现场演出），小城市用户体验明显更差。

**建议**：
- MVP 明确聚焦一线/新一线城市，不为了覆盖而做稀薄的全国版本
- 在卡片元数据中加入「城市适配性」字段，低密度城市自动过滤不适用卡片
- 设计一组「低密度友好」卡片类别：户外自然、随处可做的内在探索

**现有对应设计**：卡片设计文档未考虑地理变量。❌ 缺失。

## 二、增长与传播

### V1. 城市探索进度分享 🟡 `[ 克制版 ]` ✅ 做

**设计**：
- 「我已探索了这座城市的 X%」一键生成分享图
- 「你和 N 个探索者一起点亮了这条街」——社区归属感，不是竞争
- 不做城市间/用户间排行榜；进度只展示自己的，不做横向比较

**为什么合适**：用户在分享自己真实的探索记录，是内驱的自我表达，不是被机制征召。地图截图视觉感强，天然适合小红书/朋友圈。

---

### V2. 精准推荐分享 🟢 `[ 克制版 ]`

**设计**：打卡后可以把这张卡「推给一个朋友」，由用户自己选择要推给谁（去掉 App 主动推荐联系人的部分，避免「被分析」感）。感觉像朋友的个人推荐，而不是广告。

**实施注意**：通讯录权限只在用户主动点击「推给朋友」时才申请，不提前获取。

---

### V3. 暂缓项存档

| 项目 | 暂缓原因 | 未来重议条件 |
|------|----------|-------------|
| 挑战/点名机制 | 社会压力驱动，制造义务感 | 社区体量成熟，用户关系足够自然后，以「邀请一起做」代替「挑战」 |
| 先行者/早鸟碎片 | 不外显，避免稀缺焦虑 | 碎片序号内部存储，择机以非竞争形式呈现 |
| 季节性运营 campaign | 运营复杂度高，倒计时和限定稀缺与产品调性冲突 | 季节内容（节气卡片）已保留，campaign 随产品成熟后再议 |

## 三、架构层面

### A1. 状态机边界情况 🔴 `[ 保留 ]`

| 场景 | 问题 | 建议方案 |
|------|------|----------|
| 卡片池耗尽 | 特定兴趣用户快速刷完 | 触发「扩展推荐」，推邻近维度卡片 |
| 离线模式 | 网络相关功能降级 | 离线优先架构，预缓存队列卡片 |
| 多设备冲突 | 两台设备状态不一致 | last-write-wins + 本地操作日志 |

**现有对应设计**：状态机文档定义了正常流程，边界未覆盖。❌ 缺失。

---

### A2. 六维评分防刷机制 🔴 `[ 保留 ]`

- 打卡完成对维度分影响是右滑的 5-10 倍
- 30 天半衰期，不活跃时分数向均值回归
- 只展示增量和趋势，不展示绝对值

**现有对应设计**：评分系统文档有公式，无防刷和衰减设计。❌ 缺失。

---

### A3. 隐私设计 🔴 `[ 保留 ]`

- **本地优先永远可用**：不强制上云，数据可完全留在设备
- **导出接口 Phase 0 预留**：数据结构从最初就预留导出能力
- **敏感数据边界**：低社交分/健康分不做推断性展示，不做跨用户对比
- **合规**：收集行为和情绪数据需考虑 PIPL 合规要求

**现有对应设计**：技术架构文档提到本地存储，但隐私原则未系统化。⚠️ 部分缺失。

---

### A4. Service Interface 层补充 🟡 `[ 保留 ]`

```typescript
interface ServiceConfig {
  cachePolicy: 'no-cache' | 'cache-first' | 'stale-while-revalidate';
  offlineCapability: 'required' | 'preferred' | 'none';
  optimisticUpdate: boolean;
}
```

- 卡片队列：`stale-while-revalidate` + `offlineCapability: required`
- 维度分数：`optimisticUpdate: true`（打卡后立刻视觉反馈）
- 碎片生成：`offlineCapability: none`（降级为排队生成）

**现有对应设计**：接口定义存在，缺 Cache 和 Offline 声明。❌ 缺失。

---

### A5. AI 卡片生成质量控制 🟡 `[ 保留 ]`

- AI 生成卡片先进「待审池」，人工审核后上线
- 建立「编辑腔调」规范（温暖、不催促、有点文学性），作为 prompt 约束
- ✅ 「找一家你从没进去过的小店，买一样你不需要的小东西」
- ❌ 「立即行动！探索你的城市，提升你的生活质量！」

**现有对应设计**：AI 文档提到生成能力，未设计质量控制流程。❌ 缺失。

---

### A6. 预演文字规模化 🟡 `[ 保留 ]`

- 62 张手写预演可行；2000+ 张时需 AI 接管
- 单张卡完成 50 次后，自动生成初版 AI 预演供人工确认
- 从现有手工预演提炼 prompt 模板（视角、语气、长度、禁用词）

**现有对应设计**：碎片收藏体系文档有预演设计，规模化路径未设计。❌ 缺失。

---

### A7. 深链策略 🔴 `[ 保留 ]`

Phase 0 就要设计好 URL 结构：

```
vibe://card/{cardId}           # 查看一张卡片
vibe://checkin/{checkinId}     # 查看打卡记录
vibe://map/{city}              # 打开城市地图
vibe://fragment/{fragmentId}   # 查看碎片详情
```

**现有对应设计**：❌ 完全缺失。

---

### A8. 埋点与数据基础设施 🔴 `[ 保留 ]`

核心事件（Day 1 接入 Amplitude 免费版）：

| 事件名 | 触发时机 |
|--------|----------|
| `card_shown` | 卡片出现在视图 |
| `card_swiped` | 右滑加入队列 |
| `card_dismissed` | 左滑跳过 |
| `card_queued` | 加入计划队列 |
| `checkin_started` | 点击打卡 |
| `checkin_completed` | 完成打卡 |
| `preview_opened` | 打开预演文字 |
| `fragment_shared` | 分享碎片 |

卡片的 6 个维度标签需随事件一起上报，才能做卡片级效果分析。

**现有对应设计**：❌ 完全缺失。

---

### A9. 通知权限时机 🟡 `[ 保留 ]`

**建议时机**：用户完成第一次打卡之后——此时已产生情绪价值，接受率最高。

**绝对不在以下时机申请**：Onboarding 中、第一次右滑时、冷启动后立刻。

**现有对应设计**：用户旅程文档未提及。❌ 缺失。

---

### A10. 商业模式 🟡 `[ 暂缓，最小预留 ]`

**现阶段只做一件事**：在卡片数据结构中预留两个字段，不实现逻辑：
- `source`: `official` / `ugc` / `partner`
- `access`: `free` / `premium`

**红线**：不做广告。这不是商业决策，而是产品定义——广告与「不催促、不焦虑」的调性根本冲突。

**未来方向留档（不展开）**：纯订阅 / 本地商家合作 / 匿名数据智能 To B

## 四、产品命名方案

> 这是目前唯一没有答案的基础决策，影响调性、人群认知、ASO。
> 以下按概念方向分组，每个名字附核心主张和传播潜力判断。

---

### 方向一：「时间与积累」——你在把流逝的时光变成可见的东西

**拾光**（Shí Guāng）
- 拾 = 捡起，光 = 光，但「拾光」谐音「时光」。捡起那些被忽略的时光。
- 核心意象：地上有光，你弯腰捡起来。
- 传播句：「把生活里的光，一片一片捡起来。」
- 潜力：极高。汉字双关干净，书写美，适合所有年龄段，不挑人群，小红书天然亲和。

**余光**（Yú Guāng）
- 余光 = 眼角余光，也是剩余的光。日子里你没有认真看见的那些东西。
- 略带诗意的忧伤感，适合文艺向用户。
- 传播句：「用余光看见的，往往是真实的生活。」
- 潜力：中高。调性偏文艺，受众相对窄。

---

### 方向二：「空间与余白」——给生活留出呼吸的地方

**余白**（Yú Bái）
- 设计/书法里的负空间。好的作品需要留白，好的生活也是。
- 核心主张：不是填满，是留出。
- 传播句：「生活不是要填满，而是要留出余白。」
- 潜力：高。视觉系用户和设计圈高度共鸣，小红书极度适配，但大众渗透率相对慢。

**寻常**（Xún Cháng）
- 寻常 = ordinary，但这个 App 在重新定义寻常的价值。
- 反叙事张力：取名「寻常」，但做的是让寻常不寻常的事。
- 传播句：「寻常日子，也可以很好看。」
- 潜力：中高。有思想深度，但传播力依赖内容质量。

---

### 方向三：「探索与漫游」——城市是你的游乐场

**晃**（Huǎng）
- 单字，极简。晃 = 闲逛、悠游、没有目的地走走。
- 反效率的宣言：我就是在晃。
- 传播句：「今天晃了什么？」「来晃一晃。」
- 潜力：极高，但有风险。单字 App 名极难注册，且「晃」本身有歧义（晃眼、晃荡有负面联想）。如果能拿到，非常有记忆点。

**逛逛**（Guàng Guàng）
- 叠字，亲切，口语化。逛 = 闲逛、随意浏览。
- 已被淘宝旗下产品使用，无法使用。❌ 排除。

**漫游者**（Màn Yóu Zhě）
- 漫游 = 没有目的地的旅行。
- 偏文艺，有点重。三个字作为 App 名略长。
- 潜力：中。

---

### 方向四：「生活方式」——你过日子的方式

**活法**（Huó Fǎ）
- 直接、有力。「活法」就是生活的方式。App 名本身就是一个主张。
- 传播句：「找到自己的活法。」「你的活法是什么？」
- 潜力：极高。简短有力，自带话题性，容易引起讨论（「你觉得什么是好的活法？」）。但有一定年龄感，需要视觉设计来拉年轻感。

**过法**（Guò Fǎ）
- 「过」比「活」更轻，更日常。怎么过日子。
- 和「活法」类似但更口语、更轻盈。
- 传播句：「你平时怎么过？」
- 潜力：中高。比「活法」更轻，但力度也弱一些。

**日常**（Rì Cháng）
- 极简，但几乎没有差异化，太泛。❌ 不建议。

---

### 方向五：英文/国际向

**Mosaic**
- 马赛克拼贴画。每张碎片（fragment）是一块马赛克，拼在一起是你的人生图景。
- 和产品核心机制（碎片收藏）的隐喻完美契合。
- 潜力：高，尤其适合未来出海。发音清晰，视觉感强，无歧义。

**Drift**
- 漂流、游荡。没有目的地，跟着感觉走。
- 非常符合「不催促」调性，英文里有温柔的动感。
- 潜力：高。简洁，好拼写，调性准确。但同名产品较多，需要差异化。

**Wabi**
- 来自日语「侘び寂び（wabi-sabi）」——在不完美和无常中找到美。
- 极度符合产品哲学：碎片化的、不完整的体验本身就是美的。
- 潜力：高（精准），受众高度集中于有审美的年轻城市用户。适合做高端小众切入。

**Flâneur**（或简写 Flaner）
- 19 世纪法国文学概念：城市漫游者，没有目的地在街道上游荡，观察城市的人。
- 产品灵魂的精准描述，有文化底蕴。
- 潜力：高（精准），但小众，需要用户有一定文化背景才能共鸣。原词拼写包含法语字符，简写 Flaner 更可用。

**Lore**
- 你的个人神话、你积累的故事和知识。每个打卡是你的 lore 的一部分。
- 游戏圈里「lore」指世界观和背景故事。借用这个概念：你的生活是有 lore 的。
- 潜力：中高，偏年轻/游戏向用户共鸣强。

**Folio**
- 作品集、档案。你的生活碎片是你的 folio。
- 干净、成熟，设计师和创意行业用户亲和度高。
- 潜力：中，偏小众专业。

### 综合对比

### 我的判断（供参考）

**如果追求最大传播半径**：`拾光` 或 `活法`。
- 拾光更感性，活法更理性；拾光更女性向，活法更中性。

**如果追求调性精准、高质量用户**：`余白` 或 `Wabi`。
- 愿意接受慢慢渗透，用产品本身说话。

**如果想要国际化路径**：`Mosaic`。
- 碎片/fragment 的隐喻和产品机制完美契合，全球无歧义。

**最值得冒险的选择**：`晃`。
- 单字极难注册，但如果能拿到，记忆点无与伦比。「你今天晃了什么？」自带病毒语感。
