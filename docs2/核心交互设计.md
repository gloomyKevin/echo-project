# 核心交互设计

> 一个意象，一条线，没有多余的东西。

---

## 唯一意象：星空

| 状态 | 对应 |
|------|------|
| 一颗暗星 | 你右滑了一张卡片（想做，还没做） |
| 一颗亮星 | 你做了，打了卡 |
| 星的颜色 | 打卡时选的情绪 |
| 星之间的连线 | 共享行为原子的体验 |
| 连线密集的区域 | 你的"星座"= 画像 |
| 远处微弱的光 | 其他人点亮过的星 |

暗星不催你。它就在你的天空里，等你准备好。

---

## 两个空间

App 只有两个空间，来回切换：

**地面**：卡片流。你在这里看世界给你的可能性。
**天空**：星图。你在这里看自己走过的路。

没有第三个空间。没有队列页、收藏页、个人页、设置页。Phase 0 就这两个。

---

## 地面：卡片流

### 结构

一次只看一张卡。全屏。

```
┌─────────────────────────┐
│                         │
│      [卡片主色背景]      │
│                         │
│    去一家从没去过的餐厅    │
│    不看菜单，让店员推荐    │
│                         │
│    ·  独处或结伴  ·  城市  │
│    ·  1-2小时    ·  ¥40-120│
│                         │
│    ─────────────────     │
│    「有人说：不盯菜单的    │
│     时候，你才真的在       │
│     这里。」              │
│                         │
│         ▽ 展开预演        │
│                         │
└─────────────────────────┘

← 左滑：跳过    右滑：想做 →
         ↑ 上滑：做过了
```

### 交互

| 手势 | 发生什么 | 星空里 |
|------|---------|--------|
| 右滑 | 卡片飞向右上角，化为一颗暗星 | 暗星出现在你的天空 |
| 左滑 | 卡片淡出 | 无 |
| 上滑 | 直接进入打卡流程 | — |
| 点击「展开预演」 | 卡片下半部展开完整预演文字 | — |

**右滑的动画是关键**：卡片缩小、升起、变成一个光点飞入右上角（星图入口的位置）。用户看到这个动画就理解了"我的选择去了天空"。不需要任何文字说明。

### 左滑后的一次追问

左滑后，卡片淡出的同时，底部短暂浮现两个选项（1.5秒后自动消失）：

```
不感兴趣    现在不合适
```

不点也没关系，自动消失。但如果点了，推荐引擎获得精确信号。

### 卡片上的"他人痕迹"

每张卡片底部有一行小字，不是数字，是一个声音：

> 「有人说：不盯菜单的时候，你才真的在这里。」

Phase 0 是编辑从预演文字中提炼的金句。Phase 1+ 替换为真实用户的打卡备注。

这行字的作用不是社会证明，是**让你感觉到这张卡片背后有真实的人类经验**。

## 上滑 / 打卡流程

上滑一张卡片（表示"我做过了"），或者从星图里点一颗暗星选择打卡，进入：

```
┌─────────────────────────┐
│                         │
│   换一条路回家            │
│   你做到了。              │
│                         │
│   选一个感受：            │
│                         │
│   治愈  开心  释然        │
│   疲惫  突破  平淡        │
│                         │
│   ─────────────────     │
│   [写一句话，也可以不写]   │
│                         │
│        [ 完成 ]          │
│                         │
└─────────────────────────┘
```

点击「完成」后：

1. 屏幕切到星图视角
2. 那颗暗星亮起来，带着你选的情绪对应的颜色
3. 如果附近有其他亮星（共享行为原子），连线自动出现
4. 手机轻震
5. 屏幕底部浮现一行字（根据情绪）：

| 情绪 | 回应 |
|------|------|
| 治愈 | 有些事做完就会觉得，本来就该这样。 |
| 开心 | 对吧。 |
| 释然 | 有些重量，放下的时候才发现它一直在。 |
| 疲惫 | 累了就是真的做了。休息一下。 |
| 突破 | 第一次做的事，你永远记得当时的感觉。 |
| 平淡 | 不是每次都有大感觉。但你去做了。 |

6. 停留 2-3 秒。不急。
7. 底部轻轻浮现：「还想看看别的吗？」→ 点击回到卡片流。或者用户就停在星图里看看自己的天空。

## 天空：星图

### 进入方式

右上角一个小圆点（代表天空入口）。随着暗星/亮星增多，这个小圆点会微微有光晕。

### 星图界面

```
┌─────────────────────────┐
│  ·          *     ·     │
│        ·              · │
│    ★──────★     ·       │
│     \    /        ·     │
│      ★──★    ·          │
│  ·         ·     ★      │
│       ·       ·         │
│  ·        ○         ·   │
│        ·       ·        │
│  ·   ·     ·        ·   │
│                         │
│  你的天空  ·  7颗星       │
│                         │
└─────────────────────────┘

★ = 你的亮星（已完成）
○ = 你的暗星（想做还没做）
· = 远处微弱的光（他人的星）
──= 连线（共享行为原子）
```

### 交互

- 点击一颗亮星 → 展开这次体验的记录（卡片标题 + 情绪 + 日期 + 你写的那句话）
- 点击一颗暗星 → 展开卡片 + 「去做」按钮（进入打卡）
- 双指缩放 → 看到更大的天空（包含更多他人的微弱星光）
- 长按空白处 → 如果满足条件，浮现洞察文字

### 延迟洞察（稀有事件）

| 条件 | 出现的文字 |
|------|-----------|
| 第 3 颗亮星 | 「三颗星了。这是一个开始。」 |
| 第 1 个星座形成（3+ 颗同原子的星相连） | 「这几颗星连在了一起。你好像特别喜欢 __ 。」 |
| 第 10 颗亮星 | 「你的天空开始有形状了。」 |
| 某个区域明显密集 | 「你的天空有一个最亮的角落：__。」 |
| 14 天后回访某颗星 | 「这颗星亮了两周了。现在回想那天，什么感觉？」 |

## 暗星的生命周期（替代"褪色"和"todolist"）

暗星不催你。但它有自然的生命周期：

```
右滑 → 暗星出现（正常亮度的暗星）
         │
         ├── 7天内打卡 → 变亮星 ✓
         │
         ├── 7天未动 → 暗星开始微微闪烁（不是催促，是"它还在等你"）
         │
         ├── 21天未动 → 暗星变得极暗（几乎融入背景）
         │                不删除，但不再主动出现在地面卡片流里
         │
         └── 用户随时可以在星图里找到它，点亮它
```

**设计哲学**：暗星不消失，因为"想做的事"不应该被系统删除。但它会安静下来，不再打扰你。如果某天你突然想起来，它还在。

## 整体节奏

```
第 1 次打开：
  看到第一张卡片。没有引导，没有教程。
  卡片就在那里，像一个邀请。

前 5 张卡片：
  普世共鸣类，不试图精准推荐。
  目的：建立信任 + 采集最初的信号。

第 5 张之后：
  星图入口首次轻微发光，
  进去看到可能已有 1-2 颗暗星。
  用户理解了"我的选择去了天空"。

第 1 次打卡：
  暗星变亮。情绪回应。轻震。
  「你做了第一件事。」
  用户理解了"做了 = 星亮了"。

第 3-5 次打卡：
  星星之间出现第一条连线。
  第一次画像洞察出现。
  用户开始好奇"我的天空会变成什么样"。

持续使用：
  天空越来越丰富。
  偶尔出现新的洞察。
  远处的微弱星光让人感觉不孤单。
  打开 App 的动力不是"该做任务了"，
  是"我想看看今天有什么新的可能"。
```
