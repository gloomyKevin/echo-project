# 卡片形态与预演的深度重设计

> 卡片是这个产品唯一的触点。如果卡片不对，一切都不对。

---

## 一、当前设计哪里脱节了

把上一版的卡片拆开看：

```
标题：去一家从没去过的餐厅
副标：不看菜单，让店员推荐
标签：独处或结伴 · 城市 · 1-2小时 · ¥40-120
引言：「有人说：不盯菜单的时候，你才真的在这里。」
折叠：▽ 展开预演
```

问题：

**1. 它长得像一个产品卡片，不像一个邀请。** 标题+副标+标签+折叠——这是电商详情页的结构。用户看到的是一个"信息载体"，不是一个让他心动的东西。

**2. 预演被藏起来了。** 预演是整个产品最有力量的内容——它是让"想做"变成"真的去做"的桥。把它折叠在"展开"按钮后面，等于把最好的武器锁在柜子里。

**3. 标签（时间/花费/地点）制造了理性审判。** 用户看到"1-2小时 · ¥40-120"时，大脑立刻进入评估模式：我有没有时间？值不值这个钱？理性审判一旦启动，感性的冲动就死了。

**4. 卡片和星空意象没有视觉连接。** 卡片是彩色背景的文字块，星空是光点。两者之间缺少过渡，右滑后"化为暗星"只靠动画硬接。

## 二、重新想：卡片到底应该是什么

### 参考：那些让人"想去做"的东西

**一封朋友的消息**：
> 「昨天发现公司旁边有条小路，走进去居然有家开了二十年的面馆。老板特好玩。你下次也去看看。」

你为什么会想去？因为这不是"推荐"，是一个人的真实经历触发了你的想象。

**一首诗**：
> 「走了很远的路，/ 到一个从没到过的地方，/ 什么也没发生。/ 但来过了。」

你为什么会被打动？因为它在极短的文字里创造了一个你可以代入的场景。

**Calm 的每日冥想界面**：一张图。一个标题「放手」。没有时长，没有分类，没有"展开详情"。你看到的就是全部。点不点开，取决于这两个字有没有击中你。

**共同点：它们都不解释自己，它们直接触发想象。**

### 卡片应该是一段"微叙事"

不是标题+描述+标签。是**一段你可以在三秒内代入的文字**。

它更接近诗，而不是产品说明。

## 三、新的卡片结构

### 一张卡片 = 三层，由浅入深，自然展开

```
┌──────────────────────────────────┐
│                                  │
│                                  │
│                                  │
│    坐下来，                       │
│    告诉服务员                     │
│    「你们最拿手的给我来一份」       │
│                                  │
│                                  │
│    把决定权交出去是一件奇怪的事。   │
│    但不盯菜单的时候，              │
│    你才真的在这里。                │
│                                  │
│                                  │
│                                  │
│                                  │
└──────────────────────────────────┘
```

#### 第一层：邀请（始终可见）

不是标题，是**一个具体的动作画面**。

> ❌ 「去一家从没去过的餐厅，不看菜单让推荐」（这是标题，在描述事情）
> ✅ 「坐下来，告诉服务员'你们最拿手的给我来一份'」（这是画面，让你看见自己在做）

用户看到的第一秒，脑子里应该闪过一个画面——自己坐在那里，说出那句话。这个画面本身就是动力。

#### 第二层：预演钩子（始终可见）

邀请下方，空一行，是预演最有力量的 1-2 句话。

> 「把决定权交出去是一件奇怪的事。但不盯菜单的时候，你才真的在这里。」

不是摘要，是**能在三秒内让人产生"好像确实是这样"的那句话**。

这两层加起来就是用户滑动时看到的全部。没有标题，没有标签，没有按钮。纯文字。像一页书。

#### 第三层：完整预演 + 实际信息（上滑展开）

用户被前两层吸引，手指自然上滑（不是点按钮，是翻页的手势），卡片展开：

```
┌──────────────────────────────────┐
│                                  │
│    坐下来，                       │
│    告诉服务员                     │
│    「你们最拿手的给我来一份」       │
│                                  │
│    ──────────────────            │
│                                  │
│    可能会拿到很好的东西，           │
│    也可能就那样。                  │
│    更有意思的是，你开始             │
│    真正观察这家店——                │
│    装修是谁选的，厨师是什么路子，    │
│    桌上的东西透露了什么。           │
│                                  │
│    ──────────────────            │
│                                  │
│    约 1-2 小时 · ¥40-120         │
│    城市 · 独自或结伴               │
│                                  │
│    有人做完说：                    │
│    「第一次被推荐了一道             │
│     我绝对不会自己点的菜。真好吃。」 │
│                                  │
└──────────────────────────────────┘
```

**实际信息（时间/花费/地点）放在最底部。** 用户只有在已经被打动之后才看到它。此时它的作用不是"要不要做"的判断依据，而是"具体怎么做"的实际参考。顺序决定心理：先感性后理性，感性已经做了决定，理性只是确认可行性。

## 四、预演文字的写作结构

### 三拍结构

每段预演文字由三个节拍组成。不多不少。

| 节拍 | 作用 | 写法 |
|------|------|------|
| **遭遇** | 你会碰到什么（具体、感官） | 描述一个具体的时刻或感受，用画面而非概念 |
| **转折** | 它在某个瞬间会变 | 承认困难或不确定，然后指出转折发生的时刻 |
| **留下** | 做完后留下什么 | 不是"你会成长"，是一个具体的、做之前想不到的感受 |

### 示例对比

**「今晚上床不带手机」**

```
遭遇：头十分钟手会有点空。不是焦虑，就是习惯。
转折：第二天早上醒来——
留下：第一件事是自己的意识，不是通知。
```

**「去附近一条你路过很多次但从没进去的小路」**

```
遭遇：大概率是普通的居民区或几家小铺子。
转折：偶尔会发现意外的东西。
留下：无论如何，那条路从此在你的地图里不再是空白。
```

**「一个人去看一场你平时不会选的电影」**

```
遭遇：不熟悉的类型开头可能有点难进入。
转折：但要撑过去——通常在某一刻，你会突然理解为什么有人愿意为这种东西花两个小时。
留下：一个人看电影比你想的更好。没有人等你反应，你可以完全进去。
```

### 三拍的意义

三拍不是写作技巧，是**认知路径**：

1. **遭遇** = 降低未知恐惧（「我知道开头可能不舒服，这是正常的」）
2. **转折** = 植入实施意图（行为科学：预知困难点 + 知道它会过去 = 行动率翻倍）
3. **留下** = 提供做的理由（不是"好处"，是一个让你点头说"嗯"的感受）

这三拍合在一起，完成的事情是：**让你在做之前，已经在脑子里走了一遍。** 心理学叫"心理模拟"（mental simulation），它是除了真正做之外，最能提高行动概率的事。

## 五、卡片与星空的视觉统一

当前缺少的是：卡片（地面）和星星（天空）之间的感知连续性。

### 设计方案

**卡片的视觉底色 = 星星的颜色。**

每张卡片不用花哨的背景图。用一个**深色调底 + 微弱光晕**——就像你在夜空中看到的一颗星星的放大版。

卡片的整体视觉语言就是"夜空中的一张纸"：
- 深色背景（不是纯黑，是深蓝或深灰）
- 浅色文字
- 卡片边缘有微弱的光晕（对应这张卡片在星图中的情绪色）
- 整个 App 的色调都是"夜"

这样当卡片右滑后缩小飞入星图，视觉上是自然的——它本来就是天空的一部分，只是近距离看时能读到文字。

**意象统一**：
- 地面的卡片 = 近距离看一颗星
- 天空的星图 = 远距离看所有星
- 两者是同一个东西，只是距离不同

## 六、"他人痕迹"的重新设计

### 问题回顾

上一版用「有人说：____」放在卡片底部。问题是它和卡片主体文字在争夺注意力，且"有人说"这个框架有点刻意。

### 新方案

他人的声音不出现在卡片正面（第一二层），只出现在展开后（第三层），并且以更自然的形式：

```
有人做完说：
「第一次被推荐了一道我绝对不会自己点的菜。真好吃。」
```

这条信息出现在预演文字之后、实际信息之前。用户看到的时候已经被预演打动了，此时一个真实的人的声音起到的是**确认**作用，而非**说服**作用。

Phase 0：编辑基于预演内容模拟的声音。
Phase 1+：真实用户的打卡备注，匿名呈现。

### 星图中的他人

星图里的微弱远光不需要任何文字。纯视觉存在感。

但可以有一个微交互：**长按一颗远处的微弱星光**，浮现一行字：

> 「有人在一个雨天做了这件事。」

不是谁，不是什么时候，只是一个模糊的、有温度的事实。然后松手，文字消失。

## 七、交互手势的统一

整个 App 只有四种手势，含义一致：

| 手势 | 在卡片流 | 在星图 | 含义 |
|------|---------|--------|------|
| 左右滑 | 跳过 / 想做 | — | 选择 |
| 上滑 | 展开预演（轻）/ 做过了打卡（重） | — | 深入 |
| 点击 | — | 查看一颗星的记录 | 查看 |
| 双指缩放 | — | 拉远看更大的天空 | 视角变化 |

上滑的双重含义需要区分：**轻轻上滑 = 展开预演（卡片滑上去一部分，露出下面的内容）。用力上滑到顶 = "我做过了"，进入打卡。** 这和很多 App 的"上滑展开详情 vs 上滑到顶触发操作"是一致的，用户不需要学习。

## 八、核心流程重走一遍

带着新设计，完整走一遍用户旅程：

**第一次打开**

深色屏幕。一张卡片浮在中央。没有 logo，没有"欢迎使用"，没有引导。

```

    今晚上床的时候，
    把手机放在够不到的地方。

    头十分钟手会有点空。
    但第二天醒来，第一件事
    是自己的意识，不是通知。

```

用户的反应：「嗯。」或「这个我做不到。」

无论哪种，他会本能地滑动。

**滑了几张**

有的右滑了，看到卡片缩小、飘向右上角。右上角开始有微弱光晕。

有的左滑了。左滑后底部短暂浮现「不感兴趣 / 现在不合适」，1.5秒消失。

**上滑展开了一张**

看到完整预演。三拍结构。看完觉得"说的有道理"。然后在底部看到时间和花费，评估可行。看到一行"有人做完说"的话。

觉得可以 → 右滑。觉得还行但现在不想做 → 左滑选"现在不合适"。

**点击右上角进入星图**

几颗暗星散布在深色背景上。远处有微弱的光点。

第一反应：「哦，这些是我想做的事。」

点一颗暗星，浮现那张卡片的邀请文字。可以打卡，也可以关掉。

**第一次打卡**

选了「治愈」。暗星亮起来，蓝绿色。手机轻震。

> 「有些事做完就会觉得，本来就该这样。」

停 2 秒。然后「还想看看别的吗？」

用户要么回到卡片流，要么就在星图里待一会儿——看着自己天空里第一颗亮星。

**用了一周**

天空里 2 颗亮星，4 颗暗星。两颗亮星之间出现了一条连线。

星图底部浮现一行字：

> 「这两颗星连在了一起——你好像喜欢一个人安静地做事。」

用户看了一眼，笑了笑。或者觉得不准。但他开始好奇下一次洞察会说什么。

**这个好奇心本身，就是留存。**
