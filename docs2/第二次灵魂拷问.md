# 第二次灵魂拷问：更深的问题

> 上一次找到了飞轮断裂点和意象问题。这一次往更深处看。

---

## 一、我是一个普通用户（不是文青），我打开了这个 App

周五晚上躺沙发上刷手机。朋友发了个链接。打开。

### 第一眼：深色背景 + 大段文字

我的反应：**「这是个什么阅读 App？」**

问题出在哪：

**这个产品是 100% 文字驱动的。** 在一个抖音 3 秒定生死、小红书封面图决定点击率的时代，一个全文字的产品对用户的注意力要求极高。

不是说文字不好。而是——**"你今晚想做什么"这个状态下的用户，不想阅读**。他想要的是一个刺激、一个画面、一个"哦这个有意思"的瞬间反应。阅读是理性行为，但选择"今晚做什么"是感性行为。

### 视觉缺失是真实问题

不是说要加精美配图。而是：

**一张卡片需要在 1 秒内传递"这是什么体验"。文字做不到，画面可以。**

看两种呈现：

```
A（纯文字，现在的方案）：
「坐下来，告诉服务员'你们最拿手的给我来一份'」

B（有视觉锚点）：
[一个暖色调的模糊光斑/色块，像餐厅的柔光]
「你们最拿手的给我来一份」
```

B 不需要真实照片。一个色彩氛围 + 一句核心文字，就够了。用户 1 秒看到光的颜色就知道"这是个温暖的/户外的/安静的体验"，然后文字才跟上。

**Phase 0 方案**：不需要摄影。每张卡片用 1-2 个渐变色调表达氛围。暖橙=食物/温暖、深蓝=安静/夜晚、绿色=自然/户外、紫色=挑战/突破。这些色调同时是这张卡片在星图里的颜色——视觉统一。

## 二、核心反馈循环太慢

这是最根本的结构性问题。

### 当前循环

```
看到卡片 → 右滑（2秒）
→ 等几小时/几天 → 真的去做（几小时）
→ 回来打卡（30秒）→ 看到星亮了（2秒）
```

**反馈循环跨越了几天。** 而抖音的循环是 3 秒，Duolingo 是 5 秒。

这不是设计缺陷，是产品本质——真实体验需要时间。但问题是：**在那几天里，App 给了用户零价值。**

用户打开 App → 看新卡片 → 右滑几张 → 关掉。下次打开 → 又看新卡片 → 又右滑几张。积累了 10 张"想做"的卡片，一张都没做。

**App 变成了一个许愿池。许了愿但什么都没发生。**

### 需要一个"短循环"

在"长循环"（发现→做→打卡）之外，需要一个**每次打开 App 就能完成的短循环**。

这个短循环是什么？

**不是滑更多卡片。** 滑卡片不是价值，是获取。一直获取不行动，用户会产生负罪感。

**应该是"回顾"。** 每次打开 App，你不只是看到新的可能性，你也看到自己。

具体做法：

打开 App 时，不直接进卡片流。先停一个呼吸的时间，展示一个**极简的此刻状态**：

```
┌─────────────────────────┐
│                         │
│       你的天空           │
│       ★ 2  ○ 4          │
│                         │
│   上次做的事：            │
│   「换了一条路回家」      │
│   3 天前 · 治愈          │
│                         │
│                         │
│      [ 看看今天有什么 ]   │
│                         │
└─────────────────────────┘
```

3 秒。告诉你：你有 2 颗亮星，4 颗暗星。上次做的是什么、什么时候、什么感觉。

然后点击进入卡片流。

**这个 3 秒做了什么？**
1. 提醒你"你已经做过一些事了"——身份确认
2. 让你看到暗星数量——不是催你，是让你意识到"有东西在等我"
3. 把上次的体验拉回意识——可能触发"我今天也想做点什么"

这是一个 3 秒就能完成的短循环：打开 → 看到自己 → 决定今天要不要做点什么。

## 三、Day 2-7：用户为什么会回来？

这是任何产品最致命的问题。新鲜感过了之后，什么在拉你回来？

### 当前设计的拉力：几乎为零

- 新卡片？我已经有 10 张没做的了。
- 星图？只有 0-1 颗星，空空荡荡。
- 通知？设计原则说不催促。
- 社交？Phase 0 没有。

**一个不催你、不给你社交压力、不给你 streak 焦虑的产品，凭什么让人回来？**

### 答案不在"拉"，在"值得"

用户不回来不是因为没有被拉，是因为**回来了也没有新的价值**。

每次打开 App 都是同一个体验：看新卡片、滑、关掉。没有变化，没有惊喜，没有"只有今天才有"的东西。

需要的是：**每次打开 App，都有一点不一样。**

#### 机制 1：每日一卡（精选）

不是"今天有 15 张新卡片随便刷"。而是——

**每天只有一张精选卡，是系统基于你的画像 + 今天的时间/天气/节气选出来的。**

这张卡出现在所有其他卡片之前，有特殊的视觉标识（比如轻微的光晕动画），告诉你："这一张，是今天专门为你选的。"

卡片刷完了可以继续看更多。但精选卡是"拉你打开 App"的核心：**今天的那一张是什么？**

Calm 的 Daily Calm 就是这个逻辑。每天只有一个。你不知道今天是什么，但你知道它在等你。

#### 机制 2：天气/时间感知（免费的"AI 感"）

```
雨天打开 App →
  精选卡自动变成雨天专属：「找一家咖啡馆，带本书坐够两小时」

周五晚上 9 点打开 →
  精选卡变成夜晚微粒卡：「关灯躺着什么都不做十分钟」

清明前三天 →
  精选卡变成春日卡：「去最近的公园坐三十分钟」
```

用户不知道这是本地时间判断（不需要后端）。他只知道：**「这个 App 好像知道现在是什么天气/什么时候。」** 这种"恰好"的感觉是最强的留存。

#### 机制 3：暗星的"生命"

暗星不只是被动等待。它们偶尔"说话"。

你打开 App 时，如果有一颗暗星超过 3 天了，它可能在底部轻轻浮现一行字：

> 「'换一条路回家' 还在等你。不急。」

不是通知。不是弹窗。是在 App 内、只在你打开时出现的一句话。你看到了就看到了，不看也没关系。

这和通知推送的区别：**通知是 App 入侵你的生活。这是你主动打开 App 时，它温和地和你说了一句话。**

## 四、一个更深的问题：App 给体验本身增加了什么价值？

现在的模型：App 建议你做事 → 你自己去做 → 回来告诉 App 你做了。

**App 在"做"这一步是缺席的。** 它对体验本身没有增加任何价值。

这意味着：用户完全可以自己想到"去新餐厅吃饭"然后自己去做。App 只是一个"灵感来源 + 打卡工具"。这不够独特。

### App 应该让体验本身变得不同

怎么做到？

#### 方式 1：给每次体验一个"焦点"

不只是"去一家新餐厅"。是：

> 「去一家新餐厅。今天注意一件事：这家店用了什么颜色的灯光？」

或者：

> 「换一条路回家。数一下你路过了几家你从没注意到的店。」

一个具体的观察焦点把日常体验变成了一次"有意识的体验"。没有这个焦点，你可能走完了也没注意到什么。有了它，你的感知被放大了。

这不需要 AI。编辑在写卡片时就可以嵌入焦点。它是预演的升级版——不只告诉你"会怎样"，还告诉你"注意什么"。

#### 方式 2：打卡时追问焦点

打卡时的"写一句话"太开放了，大多数人会跳过。

如果改成一个具体的问题呢？

> 卡片焦点：「注意这家店用了什么颜色的灯光」
> 打卡时：「灯光是什么颜色？」→ 用户写：「暖黄色，特别舒服」

一个具体的问题比"写一句话"容易回答 10 倍。而且答案本身就是这次体验的独特记录——比情绪贴纸有内容得多。

这些回答积累起来，就是真正有血肉的用户数据。未来做 Agent 时，它知道的不是"你做了 10 件探索类的事"，而是"你去过的餐厅都是暖黄色灯光的，你可能喜欢有温暖感的空间"。

## 五、"上滑 = 做过了"的交互问题

在几乎所有滑动类 App 中，上滑是"super like"或"更强的肯定"。在这里变成"我以前做过了"→ 打卡流程。

**这不符合用户直觉。** 用户第一次上滑时不知道会发生什么，可能意外触发打卡流程，感到困惑。

### 重新分配手势

```
左滑 = 跳过（不变）
右滑 = 想做（不变）
点击卡片 = 展开预演（取代上滑展开）
```

**去掉上滑手势。** "做过了"这个操作不需要在卡片流里完成。用户可以在星图里找到暗星打卡，或者在展开预演后的界面里选择"我做过了"。

减少一个手势 = 降低认知负荷。两个方向（左/右）+ 点击 = 所有人都会用。

## 六、重新走一遍用户旅程（修正版）

### Day 1 第一次打开

深色背景。但不是纯文字——有一个柔和的色彩氛围。

```
┌──────────────────────────────────┐
│        [暖橙色渐变氛围光]          │
│                                  │
│                                  │
│     今晚上床的时候，               │
│     把手机放在够不到的地方。        │
│                                  │
│     头十分钟手会有点空。           │
│     第二天醒来，第一件事            │
│     是自己的意识，不是通知。        │
│                                  │
│                                  │
│                                  │
└──────────────────────────────────┘
```

左滑或右滑。卡片有色彩氛围，让决策可以靠感觉而非阅读。

### Day 1 滑了 5 张之后

右上角星图入口开始有微弱光晕。

底部浮现一行字，2 秒后消失：

> 「你好像对 __ 的事更感兴趣。」

不问"对吗？"。就是一个观察。用户如果有共鸣，信任开始建立。

### Day 2 第二次打开

不直接进卡片流。先看到：

```
你的天空  ★ 0  ○ 3
──────────
今天为你选的：
[绿色氛围光]
「去最近的公园，坐够三十分钟。」
──────────
[看看今天有什么]
```

精选卡和昨天的不同。用户知道了"每天有一张专门给我的"。

### Day 3 做了第一件事，回来打卡

选了情绪。然后看到一个具体的问题（焦点追问）：

> 「你在公园里坐了多久开始不觉得无聊了？」

用户写：「大概十五分钟」。

然后星亮了。情绪回应。停 2 秒。

> 「还想看看别的吗？」

### Day 7

打开 App：

```
你的天空  ★ 2  ○ 5
──────────
「换一条路回家」还在等你。不急。
──────────
今天为你选的：
[紫色氛围光]
...
```

用户已经形成了习惯：打开 App → 看看今天的精选是什么 → 看看自己的天空。

两颗亮星之间出现了连线。第一次洞察浮现：

> 「你做的两件事都是一个人、在户外——你喜欢独自在城市里走。」

用户的反应：「嗯，好像是。」

**这个"嗯"就是产品价值的开始。**

## 七、修正总结

| 上一版问题 | 修正 |
|-----------|------|
| 纯文字卡片，1 秒内无法传递氛围 | 加色彩氛围光，让感性直觉先行 |
| 反馈循环太慢，打开 App 无新价值 | 每日精选 + 打开时看到自己的天空状态 |
| Day 2-7 无拉力 | 每日精选（每天不同）+ 天气/时间感知 + 暗星偶尔说话 |
| App 对体验本身无增值 | 每张卡附带观察焦点 + 打卡时追问焦点 |
| 上滑手势反直觉 | 去掉上滑，只留左滑/右滑/点击 |
| 打卡"写一句话"太开放 | 改为焦点追问，具体问题更容易回答 |
