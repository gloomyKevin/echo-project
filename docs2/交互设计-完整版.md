# Vibe 交互设计完整版

> 用于在 Stitch 中生成完整交互原型的设计稿。
> 以用户视角走一遍完整体验，每一步都标注视觉、动效、手势。
> 日期：2026-03-25

---

## 全局设定

### 唯一空间

整个 App 是一片星空。所有界面共享同一片天空，区别只是"镜头"不同。

### 色彩

```
背景：#0A0E14（极深蓝黑）
文字：#E8E6E3（暖白）
弱文字：rgba(232,230,227, 0.45)
极弱文字：rgba(232,230,227, 0.2)
边框：rgba(255,255,255, 0.08)

氛围色：
  暖橙 #F59E42 — 食物、社交、温暖
  静蓝 #5B9CF5 — 安静、夜晚、内省
  生绿 #4ADE80 — 自然、户外、运动
  突紫 #A78BFA — 挑战、突破、第一次
  淡白 #C9CDD4 — 日常、微粒
```

### 字体

系统字体。正文 14-15px，邀请文字 20-22px，弱提示 13px。字重主要用 300（轻）和 400（常规）。

### 星星形态

不是圆点。是**有光芒的星**。

**亮星**（已完成的体验）：
- 中心：四角星形，大小 6-10px（按体验强度），颜色=打卡时选的情绪色
- 光芒：从中心向上下左右四个方向延伸的细线，长度=中心的 2.5 倍，半透明
- 光晕：中心向外扩散的柔和圆形渐变，半径=中心的 5 倍，极低透明度
- 动效：稳定微闪（亮度在 85%-100% 之间缓慢波动，周期 3-5 秒）

**暗星**（想做但还没做）：
- 中心：四角星形，大小 4-6px，半透明白色
- 光芒/光晕：几乎不可见
- 动效：缓慢呼吸明灭（透明度 25%-45%，周期 2-3 秒）
- 超过 7 天未完成：闪烁变得不规则，像快要熄灭的烛火

**背景星**（纯装饰）：
- 小圆点，1-2px，散布约 100-130 颗
- 透明度极低（5%-20%），有微弱闪烁
- 上方更密集更亮，下方渐疏渐淡

**星座连线**：
- 共享同一主行为原子的亮星之间自动连线
- 线宽 0.5px，透明度 10-15%
- 3 颗以上相连时，围合区域有极微弱的面积填充

## 场景 1：首次打开

### 用户看到的

全屏深色背景。背景星缓慢闪烁。没有用户星星（因为还没有任何数据）。

屏幕中央偏下，一行文字淡入（delay 0.5s，duration 0.6s）：

```
晚上好。
```

（根据时段变化：早上好 / 上午 / 午后 / 下午了 / 傍晚 / 晚上 / 深夜了）

再过 1 秒，文字下方淡入一行更淡的提示（opacity 0.15）：

```
轻触任意位置
```

### 用户操作

点击屏幕任意位置。

### 发生什么

文字淡出（0.3s）。短暂停顿（0.2s）。

第一张卡片从屏幕下方升入（y: 从 +60px 到 0，duration 0.5s，ease-out）。

进入卡片流。

## 场景 2：卡片流（核心交互）

### 画面布局

```
┌──────────────────────────────┐
│                              │
│    ·  ·    ·       ·    ·    │  ← 背景星（上方更密）
│  ·      ·     ·         ·   │
│     ·       ·    ·          │
│                              │
│  ┌────────────────────────┐  │
│  │                        │  │
│  │  [氛围色光晕]           │  │
│  │                        │  │
│  │  今晚上床的时候，       │  │  ← 卡片居中偏下
│  │  把手机放在             │  │
│  │  够不到的地方。         │  │
│  │                        │  │
│  │  头十分钟手会有点空。   │  │
│  │  第二天醒来，第一件事   │  │
│  │  是自己的意识，不是通知。│  │
│  │                        │  │
│  └────────────────────────┘  │
│                              │
│                              │
└──────────────────────────────┘
```

卡片占屏幕约 55-60% 高度，水平边距 20px。卡片上方有约 35% 的星空可见区。

### 卡片视觉

- 背景：`rgba(14, 20, 28, 0.88)` + `backdrop-filter: blur(24px)`
- 边框：1px `rgba(255,255,255,0.08)`
- 圆角：20px
- 阴影：`0 20px 60px rgba(0,0,0,0.5)`，`inset 0 1px 0 rgba(255,255,255,0.05)`
- 内部上方：氛围色的模糊渐变光晕（`radial-gradient`，从顶部 40% 处扩散）

### 卡片正面内容

```
[氛围光]

邀请文字                    ← 20-22px，字重 300，行高 1.75
（一个具体的动作画面）

                            ← 间距 24px

预演钩子                    ← 14px，色 rgba(232,230,227,0.45)，行高 1.85
（最有力的 1-2 句话）
```

### 卡片背面内容（点击翻转后）

```
完整预演文字                ← 14px，行高 1.9

────────────                ← 分割线，rgba(255,255,255,0.08)

约 1-2 小时 · ¥40-120      ← 13px，极弱色
城市 · 独自或结伴

有人做完说：                ← 13px，弱色，opacity 0.7
「第一次被推荐了一道
  我绝对不会自己点的菜。」
```

### 手势：飞升（右滑 / 上滑）= 想做

拖动卡片向右或向上。

**视觉反馈**：
- 卡片边框从默认色渐变为氛围色（随拖动距离线性过渡）
- 卡片整体亮度微增（`brightness` 从 1.0 到 1.15）
- 星空背景微微变亮（opacity 从 0.4 到 0.55）

**松手判定**：拖动距离 >80px 或速度 >400px/s 时触发飞升。否则弹回。

**飞升动画**（触发后，约 400ms）：
1. 卡片向右上方加速移出（目标位置：屏幕右上方）
2. 移动过程中卡片逐渐缩小（scale 1.0→0.3）+ 透明度降低
3. 到达星空区域后完全消失
4. 星空中对应位置出现一颗新的暗星（从 scale 0 弹出到 1，spring 动效）

**飞升后**：
- 星空区域浮现一行字（opacity 0.4，1.5 秒后淡出）：
  - 卡片 space=home → 「今晚试试。」
  - space=nearby → 「下次路过。」
  - space=city → 「找个时间。」
  - space=nature → 「等一个好天气。」
- 下一张卡片从下方升入（delay 0.3s）

### 手势：沉降（左滑 / 下滑）= 暂时不要

拖动卡片向左或向下。

**视觉反馈**：
- 卡片整体变暗（`brightness` 从 1.0 到 0.6）
- 卡片变得微透明（opacity 从 1.0 到 0.7）

**松手判定**：同飞升。

**沉降动画**（触发后，约 300ms）：
1. 卡片继续向沉降方向移动 + 向下沉
2. 透明度降到 0
3. 完全消失

**沉降后**：
- 无文案，安静消失
- 下一张卡片从下方升入（delay 0.2s）

### 手势：点击 = 翻转

点击卡片任意位置 → 卡片 Y 轴 3D 翻转到背面（duration 0.25s）。

翻转后仍可拖动飞升/沉降。再点一次翻回正面。

翻转时卡片微微浮起（scale 1.0→1.02→1.0），增加"翻动"的手感。

### 首次引导

**第 1 张卡片**：
- 出现后静止 1.5 秒
- 卡片自动微微向右上方漂移 6px 再弹回（暗示方向），spring 动效
- 同时卡片下方出现极淡文字（opacity 0.18，2.5 秒后淡出）：

  ```
  向上或向右 · 想做
  ```

- 如果 5 秒内未点击，卡片微抖动一次（暗示可翻转）

**第 2 张卡片**：
- 只有微漂移，无文字

**第 3 张起**：
- 无任何引导

### 进入星图

卡片上方的星空区域是可点击的。点击星空（屏幕上方约 35%）→ 进入星图全景。

或点击右上角呼吸点（有星星后出现）。

**过渡动效**：
- 卡片向下沉出屏幕（y: 0→+screen height，duration 0.4s）
- 同时星空亮度提升（opacity 0.4→0.2，即背景星变暗但用户星星变亮 0.3→1.0）
- 用户星星放大变清晰
- 整体是"镜头向上抬起，看到完整天空"的感觉

## 场景 3：信笺（卡片流中的系统对话）

### 触发时机

- 累计 5/10/15 次选择后 → 画像确认
- 每天第一张卡之前 → 精选预告（如果有精选卡）
- 第 1/3/5/10 次打卡后回到卡片流 → 里程碑

### 信笺视觉

和体验卡片共享形态，但有区分：

```
┌ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┐   ← 虚线边框


     你好像更喜欢                ← 16px，字重 300，弱色
     「探索」类的体验。            居中


└ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┘
```

区别于体验卡片：
- **虚线边框**（dashed）代替实线
- **无氛围光**（或极淡白光）
- **背景更透明**：rgba(14,20,28, 0.65)
- **文字居中**，字号稍小

### 信笺交互

- 向任意方向轻推即消失（阈值比体验卡低，40px 即可）
- 有选项的信笺：底部有 2-3 个选择按钮，点击后消失
- 消失后下一张体验卡片正常升入

## 场景 4：精选卡

### 视觉区分

精选卡是当天第一张出现的体验卡片。和普通卡片的区别：

- 边框更亮：`rgba(255,255,255, 0.2)` 代替 0.08
- 氛围光有**缓慢脉动**（亮度在 60%-100% 之间，周期 4 秒）
- 卡片右上角有一颗极小的四角星标记 ✦（opacity 0.5，12px）
- 外部有一圈极微弱的光晕（`box-shadow: 0 0 30px rgba(氛围色, 0.08)`）

其他交互（翻转、飞升、沉降）和普通卡完全一致。

## 场景 6：打卡

### 进入方式

星图中点击暗星 → 「★ 我做了」。

### 步骤 1：选情绪

```
┌──────────────────────────────┐
│                              │
│    （星空背景，微弱）          │
│                              │
│                              │
│      今晚上床的时候，         │  ← 18px，字重 300
│      把手机放在够不到的地方。  │
│                              │
│                              │
│   [ 治愈 ] [ 开心 ] [ 释然 ]  │  ← 3×2 网格
│   [ 疲惫 ] [ 突破 ] [ 平淡 ]  │
│                              │
│                              │
└──────────────────────────────┘
```

点击某个情绪 → 该按钮边框变为对应颜色 → 0.2 秒后过渡到步骤 2。

### 步骤 2：焦点追问

```
┌──────────────────────────────┐
│                              │
│          治愈                 │  ← 情绪色，15px
│                              │
│  「第二天醒来第一个念头        │  ← 焦点问题，15px
│    是什么？」                 │
│                              │
│  ┌──────────────────────┐    │
│  │写一句话，也可以跳过    │    │  ← textarea
│  └──────────────────────┘    │
│                              │
│       [ ★ 点亮 ]             │  ← 按钮
│                              │
└──────────────────────────────┘
```

点击「★ 点亮」→ 手机轻震 → 进入打卡完成。

### 打卡完成

**过渡动效**（整体约 2 秒）：

1. 当前界面淡出（0.3s）
2. 星空过渡到星图全景参数（背景星变暗，用户星星变亮）（0.4s）
3. 刚才的暗星亮起：
   - 从暗白色变为情绪色
   - 四角星芯出现
   - 十字光芒从零展开
   - 光晕脉冲扩散一次
   - spring 弹性动效，整个过程 0.8s
4. 如果附近有其他同原子的亮星 → 连线渐现（0.5s）
5. 屏幕下方浮现情绪回应文案（delay 0.8s，opacity 0.6）：

| 情绪 | 文案 |
|------|------|
| 治愈 | 有些事做完就会觉得，本来就该这样。 |
| 开心 | 对吧。 |
| 释然 | 有些重量，放下的时候才发现它一直在。 |
| 疲惫 | 累了就是真的做了。休息一下。 |
| 突破 | 第一次做的事，你永远记得当时的感觉。 |
| 平淡 | 不是每次都有大感觉。但你去做了。 |

6. 文案停留 3 秒后淡出
7. 再过 2 秒，如果用户无操作 → 自动过渡回卡片流（星空参数回卡片流状态，卡片从下方升入）
8. 如果用户在星图中点击/交互 → 停留在星图，不自动退出

### 延迟洞察

在星图全景中，当满足条件时，星图中浮现一段文字（中下方，opacity 0.5，停留至用户离开星图）：

| 条件 | 文案 |
|------|------|
| 第 3 颗亮星 | 三颗星了。这是一个开始。 |
| 首次形成星座（3+ 颗同原子亮星相连） | 这几颗星连在了一起。你好像特别喜欢____。 |
| 第 10 颗亮星 | 你的天空开始有形状了。 |
| 某区域明显密集 | 你的天空有一个最亮的角落：____。 |

## 场景 7：再次打开 App（非首次）

### 有数据时的着陆

```
┌──────────────────────────────┐
│                              │
│    ·  ★  ·    ○    ·    ·    │  ← 用户星星可见（弱）
│  ·      ·  ★──★     ·       │
│     ·       ·    ·          │
│                              │
│                              │
│      晚上。                   │  ← 情境文案
│                              │
│  「换一条路回家」还在等你。    │  ← 状态文案（弱色）
│                              │
│                              │
│        轻触任意位置            │  ← opacity 0.15
│                              │
└──────────────────────────────┘
```

星空中用户星星微弱可见（userOpacity 0.6），提醒你"你的天空有东西了"。

**状态文案优先级**：
1. 有暗星 >7 天 → 「"____" 还在等你。」
2. 有上次打卡记录 → 「上次：「____」· 3 天前 · 治愈」
3. 连续 5 天未打开 → 「好久不见。你的天空还在。」
4. 都不满足 → 不显示状态文案

点击任意位置 → 同首次，过渡到卡片流。

### 呼吸点

右上角，一个 5px 的小圆点。有亮星后外圈出现呼吸光晕（radial-gradient，蓝色，4 秒周期缓慢膨胀收缩）。

两种入口效果一样。

## 视图间过渡汇总

所有过渡的本质是**同一片星空的参数变化**：

| 过渡 | 背景星亮度 | 用户星亮度 | 卡片 | 时长 |
|------|-----------|-----------|------|------|
| 着陆 → 卡片流 | 0.7→0.4 | 0.6→0.3 | 从下方升入 | 0.5s |
| 卡片流 → 星图 | 0.4→0.2 | 0.3→1.0 | 向下沉出 | 0.4s |
| 星图 → 卡片流 | 0.2→0.4 | 1.0→0.3 | 从下方升入 | 0.4s |
| 卡片流 → 打卡 | 0.4→0.3 | 0.3→0.2 | 淡出 | 0.3s |
| 打卡 → 星图（点亮） | 0.3→0.2 | 0.2→1.0 | 无 | 0.4s |
| 星图 → 卡片流（自动） | 0.2→0.4 | 1.0→0.3 | 从下方升入 | 0.5s |

所有参数用 spring 或 ease-out 过渡，用户感知到的是**镜头在同一个空间里平滑移动**。
