# 第一性推演：从人性到设计

> 不堆功能，不堆意象。从人到底需要什么开始，推导出设计应该是什么样的。

---

## 一、重新审视上一份文档的问题

上一份文档犯了一个错误：**用产品经理的逻辑去修补用户的人性。**

"什么时候做？" → todolist 化。用户右滑的那一刻是轻盈的、充满可能性的。紧接着问"你打算什么时候做"，就像你说"好想去旅行啊"的时候旁边有人递过来一张行程表。魔法瞬间被打碎了。

"72% 的人选了治愈" → 冰冷的统计。你做完一件事，心里有某种说不清的感觉，然后屏幕上出现一个百分比。这就像你刚看完一部让你流泪的电影，旁边有人说"这部电影在豆瓣 8.7 分"。它在用理性的语言回应一个感性的时刻。

**根本问题：上一份文档在试图用"机制"解决"感受"。但感受不能被机制解决，只能被设计唤起。**

## 二、回到第一性：人到底在什么时候会真的去做一件事？

### 行为科学给出的答案

BJ Fogg 的行为模型：**行为 = 动机 × 能力 × 触发，三者同时满足的那一刻。**

这意味着：
- 不是"提前计划了就会做"（todolist 逻辑）
- 而是"在对的时刻、对的状态下、看到了对的东西"

换句话说：**App 不应该帮用户规划行动，而应该在用户最可能行动的那个时刻出现在他面前。**

这和 Tinder 的逻辑一样——Tinder 不会问你"打算什么时候去约会"。它在你无聊的时候出现，让你滑，match 自然发生。**行动的发生不是被计划的，是被时机催化的。**

### 那些真的改变了人行为的产品是怎么做的

**Forest（专注森林）**：不告诉你"该专注了"。它只是种了一棵树，你知道如果你离开，树会死。行为的触发不是提醒，是**一个你不忍心破坏的东西**。

**Duolingo 的猫头鹰**：研究表明它的 streak 机制本身不是留存的根因。真正起作用的是那只绿色猫头鹰——一个有"情感重量"的角色。你不是在"完成今日任务"，你是在"不让它失望"。这是一个微小的情感连接。

**Journey（游戏）**：你在广袤沙漠中独行，突然遇到另一个玩家。你们不能说话，只能发出音符。你们一起走了一段路，然后分开。没有好友列表，没有聊天。但几乎每个玩家都说这是他们玩过最感动的体验。**"你并不孤单"不需要社交功能，只需要让人感知到他人的存在。**

**Dark Souls 的地上留言**：前一个玩家在危险的地方写下"小心前方"或者"不要放弃"。你看不到这个人，但你知道有人在你之前走过这条路，他特意留了一句话。这种**匿名的、异步的、有温度的存在感**，比任何社交功能都更深刻。

**Spotify Wrapped**：它不在你听歌的时候打扰你。它等了整整一年，然后在年底告诉你"你今年是前 0.5% 的 Radiohead 听众"。你的反应是惊喜——**"原来我是这样的人"**。这就是画像的终极价值：不是数据，是自我发现的惊喜。

**Pokémon GO 的"附近"雷达**：不告诉你"去这里抓"。只是在右下角显示附近有什么。你知道它在那里。当你恰好路过的时候，你自然会打开 App。**不是推你去做，是让你知道可能性就在旁边。**

## 三、从这些洞察中推导出的设计原则

### 原则 1：不要计划，要渗透

App 不应该帮用户"安排"体验，而应该**渗透进用户生活的自然缝隙里**。

右滑一张卡片之后，它不进入 todolist。它变成一颗**种子**——安静地待在那里，等待合适的时刻。

什么时候这颗种子会发芽？当用户下次打开 App，发现首页不再是新卡片，而是：

> 你上次想做的事，还记得吗？「换一条路回家」

不问什么时候。不催。只是静静地在那里。**它的存在本身就是触发。**

如果用户今天下班恰好想走不同的路，他会想起来。如果不想，他划过去看新卡片。没有任何压力。

这就是 Forest 的逻辑——树不催你专注，它只是在那里生长。

### 原则 2：反馈不是数据，是镜子

打卡后用户需要的不是"分数变化"或"统计数据"，而是**一面镜子——让他看到一个自己没注意到的自己**。

最好的镜子是什么样的？

**不是实时的。** 实时反馈太快，没有积累感。就像你每天称体重不如一个月后看到对比照。

**是延迟的、意外的、有发现感的。** 像 Spotify Wrapped 那样——在你没有预期的时候，告诉你一件关于你自己的、你没注意到的事。

设计推导：
- 打卡后的即时反馈应该是**情感性的**，不是数据性的——一个美丽的动画、一颗星亮起来、一段回应你选择的情绪的短文字
- 画像洞察应该是**延迟的、稀有的**——做了 5 件事之后，第一次出现一段洞察；做了 10 件之后出现第二次。频率低，每次都有分量
- 维度分数可以存在，但藏在深处，是给愿意看数据的用户的——不是核心反馈

### 原则 3：一个意象，贯穿始终

你的直觉是对的——不能堆叠意象。地图、星空、碎片、维度雷达图……每多一个视觉隐喻，产品就分裂一次。

**应该选一个，让它承载一切。**

## 四、一个意象的选择

### 候选意象分析

| 意象 | 优势 | 局限 |
|------|------|------|
| **地图点亮** | 和"探索"强绑定，城市感强，可分享 | 大量卡片（内省/居家类）无地理属性，覆盖不了全部体验 |
| **星空/星座** | 万物可承载（不依赖地理），美感强，私密感好 | 和"走出去做事"的行动感有距离 |
| **花园/森林** | 生长隐喻好，Forest 已验证 | 偏可爱，和"认识自己"的深度感有冲突 |
| **拼图/马赛克** | 每块碎片拼出完整画面 = 认识自己 | 视觉上偏静态，缺少生长感 |

### 我的推导

如果从这个产品的终极命题出发——**"帮你认识自己"**——那最好的意象不是外部世界（地图、花园），而是**内在世界的可视化**。

但"认识自己"太抽象了。需要找到一个视觉隐喻，让它既有美感、又有生长感、又能承载所有类型的体验。

**星图（Star Chart）可能是最合适的。**

理由：

1. **每一次真实体验 = 点亮一颗星。** 不依赖地点，在家不带手机也是一颗星，去陌生城市也是一颗星。
2. **星星之间会形成连线 = 你的行为模式。** 多次选择探索类体验，那个区域的星星密集，自然形成"星座"。星座就是你的画像，但用户看到的不是数据，是一幅属于自己的星图。
3. **别人的星星微弱可见 = "你并不孤单"。** 你的星星明亮，他人的星星是远处微弱的光。你不知道是谁，但你知道那里有人。Journey 的感觉。
4. **星空自然是美的。** 不需要游戏化设计，星空本身就值得看、值得分享。
5. **星空 → 认识自己。** 人类仰望星空的本质行为就是在追问"我是谁"。这个意象和产品终极命题是统一的。
6. **空的星空也不丑。** 新用户的空星空不会让人感到匮乏，因为夜空本来就大部分是黑暗的。而一个空的地图、空的花园会让人觉得"我什么都没做"。

### 星图怎么运作

```
打开 App → 首页是卡片流（做事的地方）
↓
点一下某个角落 → 进入你的星图（看自己的地方）
↓
星图里每颗亮星 = 一次完成的体验
星的颜色 = 打卡时选择的情绪（治愈=蓝绿，突破=橙红，平淡=淡白……）
星的大小 = 体验的强度层级
星之间的连线 = 共享行为原子的体验自然相连
形成的区域 = 你的"星座"（探索者、连接者、内省者……）
远处微弱的星光 = 其他用户点亮的星

偶尔（做了 5 件事 / 10 件事 / 某个里程碑），
星图上浮现一段话——
「你的星空里，独自完成的体验占了 70%。你是一个享受独处的人。」
```

**这一个意象，承载了：完成反馈、画像展示、成长感、社区存在感、自我发现。** 不需要额外的维度雷达图、碎片收藏、地图点亮。一个就够。

## 五、重新设计飞轮：不打断、不催促、自然转动

### 打开 App 时

不是空的发现页。首先看到的是：

**如果有一张"进行中"的卡片**（上次右滑的最新一张）：
> 卡片安静地浮在那里，和新卡片栈并存。不挡路，不催促。用户可以点它打卡，也可以忽略它去看新卡片。

**如果没有进行中的卡片**：
> 直接进入新卡片流。

这和"渗透"原则一致——你想做的事不在一个 todolist 里，它就在你的视野里，像床头放的那本书。

### 右滑时

卡片飞向屏幕上方某个位置（或者变成一颗暗星飘向星图方向），表示"它记住了"。

不问任何问题。不问什么时候做。**轻盈是右滑最重要的品质。**

### 做了之后打卡

极简：情绪（必选，6 选 1）→ 一句话（可选）→ 完成。

完成的瞬间：
1. 屏幕上，那颗暗星变亮，带着你选择的情绪的颜色，缓缓飘向星图
2. 手机轻震
3. 一行字，不是数据，是回应：

根据情绪选择返回不同的文字：
- 治愈 → 「有些事情做完就会觉得，本来就该这样。」
- 突破 → 「第一次做的事，你永远记得当时的感觉。」
- 平淡 → 「不是每次都有大感觉。但你去做了，这就是全部。」
- 开心 → 「对吧。」
- 释然 → 「有些重量，放下的时候才发现它一直在。」
- 疲惫 → 「累了就是真的做了。休息一下吧。」

这些文字是固定的（Phase 0），但它们承载着情感重量。不是"恭喜你完成了！"，是**一个懂你的朋友在你做完一件事后说的那一句话**。

4. 然后沉默 1 秒。不急着推下一张。让这个时刻停一停。

5. 1 秒后，底部轻轻浮现一行小字：

> 「还有一件你可能想做的事——」
>
> （一张和刚完成的体验共享某个行为原子的新卡片，从下方升起）

用户可以看，可以右滑，也可以关掉 App。飞轮的种子已经种下了。

### 星图里的延迟洞察

不是每次打卡都给洞察。太频繁就廉价了。

| 触发条件 | 洞察内容 |
|---------|---------|
| 第 3 次打卡 | 「你的星空亮了 3 颗星。这是开始。」 |
| 第 5 次打卡 | 第一次画像洞察：「你做的事情里，__ 占了大多数。」 |
| 连续 2 次相同行为原子 | 「你好像特别喜欢 __ 类的体验。再来一个？」 |
| 第 10 次打卡 | 第一次形成"星座"：「这些体验连在一起，看起来像一个 __ 的人。」 |
| 第 1 次打卡后 14 天 | 回访：「两周前你做了 __。现在回想起来，什么感觉？」 |
| 某个维度明显高于其他 | 「你的星空有一个明亮的角落：__。」 |

每次洞察都是稀有的、有分量的。用户会期待下一次洞察——这本身就是留存。

## 六、"你并不孤单" — 用 Journey 的方式而不是社交的方式

Phase 0 不需要后端也能做到：

### 卡片上

不写"已有 2847 人做过"（冰冷的数字）。

改为：

> 「有人在深夜做了这件事，说'比想的容易'。」

Phase 0 这是编辑写的。Phase 1+ 替换为真实用户的打卡备注（匿名脱敏）。

用户看到的不是一个数字，是**一个匿名的、但有体温的声音**。Dark Souls 的留言机制。

### 星图上

你的星星明亮。
远处有其他人的星星，很微弱、很远，但在那里。
你不知道是谁。你只知道有人和你在同一片天空下。

Phase 0 可以用算法随机生成背景微光来模拟。Phase 1+ 替换为真实用户数据。

## 七、汇总：一个克制的、完整的产品

```
核心循环：
看卡片 → 预演（想象这件事） → 右滑（想做） → 去做 → 打卡 → 星亮了 → 看下一张
                                                              ↓
                                                    偶尔，星图里浮现一段洞察
                                                    「你比你以为的更 __。」

核心意象：只有一个——星图
核心反馈：只有一种——星亮了 + 一句回应
核心留存：不是 streak，不是通知，是「我想看看我的星空会变成什么样」
核心的"不孤单"：远处那些微弱的星光

不做的事：
- 不做 todolist（不问什么时候做）
- 不做排行榜
- 不做 streak
- 不做弹窗通知催促
- 不做维度雷达图（数据藏在星座里）
- 不做多个意象（不同时有地图+星空+碎片+维度图）
```

这个产品的克制程度应该像 **Calm** 而不是 **Duolingo**。它不追着你跑，它等你来。但每次你来，你都能发现一点新的东西——关于世界，关于自己。
