# Vibe v0.1 需求文档

> 版本：0.1（H5 原型）
> 目标：验证核心假设——卡片能不能让人真的去做一件事
> 载体：H5 页面，Vercel 部署，分享链接给 10-20 个朋友
> 日期：2026-03-24

---

## 一、产品定位

一句话：**推你去做一件你不知道自己会喜欢的事，然后通过你真正做了什么来帮你认识自己。**

不是攻略平台，不是 todolist，不是性格测试。
是一个帮你"在场"的工具。

### v0.1 验证的核心假设

**假设：大多数人不是不想让生活更丰富，是缺少一个"推一把"。**

验证指标：**打卡率 > 15%**（右滑后真的去做了并回来打卡的比例）。

## 二、产品结构

### 两个空间，没有第三个

| 空间 | 是什么 | 做什么 |
|------|--------|--------|
| **地面** | 卡片流 | 看世界给你的可能性 |
| **天空** | 星图 | 看自己走过的路 |

没有：个人页、设置页、收藏页、队列页。

### 整体页面结构

```
┌─────────────────────────┐
│  [着陆屏]                │  ← 打开 App 先看到
│  你的天空状态 + 今日精选   │
├─────────────────────────┤
│  [卡片流]                │  ← 核心交互区
│  一次一张，左滑/右滑/点击  │
├─────────────────────────┤
│  [打卡流]                │  ← 完成体验后
│  情绪 + 焦点追问          │
├─────────────────────────┤
│  [星图]                  │  ← 右上角入口
│  亮星/暗星/连线/洞察      │
└─────────────────────────┘
```

## 四、卡片流（地面）

### 4.1 卡片视觉

整体色调：深色（深蓝灰 #0F1419 为主背景）。卡片浮在深色背景上。

每张卡片有一个**氛围色**（柔和渐变光晕），传递感性印象：

| 氛围色 | 对应体验类型 |
|--------|------------|
| 暖橙 | 食物、温暖、社交 |
| 深蓝 | 安静、夜晚、内省 |
| 绿色 | 自然、户外、运动 |
| 紫色 | 挑战、突破、第一次 |
| 浅白 | 日常、微粒、随手可做 |

氛围色 = 这张卡在星图里亮起来后的颜色。

### 4.2 卡片内容结构（三层）

#### 第一层 + 第二层（始终可见，滑动时看到的全部）

```
┌──────────────────────────────────┐
│        [氛围色渐变光晕]            │
│                                  │
│                                  │
│     坐下来，                      │
│     告诉服务员                     │
│     「你们最拿手的给我来一份」      │   ← 第一层：邀请
│                                  │      一个具体的动作画面
│                                  │
│     把决定权交出去是一件奇怪的事。  │   ← 第二层：预演钩子
│     但不盯菜单的时候，             │      最有力量的 1-2 句话
│     你才真的在这里。               │
│                                  │
│                                  │
└──────────────────────────────────┘
```

**第一层（邀请）写法要求**：
- 不是标题（❌「去一家从没去过的餐厅」），是画面（✅「坐下来，告诉服务员'最拿手的来一份'」）
- 用户看到的第一秒脑子里应该闪过一个自己在做这件事的画面

**第二层（预演钩子）写法要求**：
- 预演中最有冲击力的 1-2 句
- 不是摘要，是能在 3 秒内让人产生"好像确实是这样"的那句话

#### 第三层（点击卡片后展开）

```
┌──────────────────────────────────┐
│     [第一层 + 第二层不变]          │
│                                  │
│     ──────────────────           │
│                                  │
│     [完整预演]                    │
│     三拍结构：遭遇→转折→留下       │
│     或其他预演形态                 │
│                                  │
│     ──────────────────           │
│                                  │
│     约 1-2 小时 · ¥40-120        │  ← 实际信息，最后出现
│     城市 · 独自或结伴              │
│                                  │
│     有人做完说：                   │  ← 他人痕迹
│     「第一次被推荐了一道            │
│      我绝对不会自己点的菜。」       │
│                                  │
└──────────────────────────────────┘
```

### 4.3 预演的 6 种形态

每张卡片选最适合的 1 种：

| 形态 | 适用场景 | 示例 |
|------|---------|------|
| **经典叙事**（三拍：遭遇→转折→留下） | 安静、内省类卡片 | 「头十分钟手会有点空。第二天醒来，第一件事是自己的意识。」 |
| **一个事实** | 有数据/冷知识支撑的卡片 | 「盲点菜在日本叫 omakase。米其林三星几乎都这样。你家楼下也可以。」 |
| **多人声音** | 大众类、社交类卡片 | 做过的人说—— 「比想的简单」「老板聊了半小时」「那条街啥也没有哈哈」 |
| **一个问题** | 观察类、探索类卡片 | 「你能说出公司和家之间的路上有几家店吗？」 |
| **挑战规则** | 高行动力、游戏感的卡片 | 「规则：坐地铁到终点。不查地图。出站，走。」 |
| **极短** | 低门槛微粒卡 | 「就发。」「今天买。」 |

### 4.4 交互手势

| 手势 | 效果 |
|------|------|
| **右滑** | 想做 → 卡片缩小、化为光点飞向右上角星图入口 → 暗星出现 |
| **左滑** | 跳过 → 卡片淡出 → 底部浮现「不感兴趣 / 现在不合适」1.5 秒后自动消失 |
| **点击卡片** | 展开第三层（完整预演 + 实际信息 + 他人痕迹） |

展开状态下可右滑/左滑，效果同上。

### 4.5 每日精选

每天第一张卡是系统精选，有特殊视觉标识（光晕动画更明显）。

精选逻辑：
1. 节气窗口内 → 节气卡强制为精选
2. 天气反差命中（连续阴雨后放晴等） → 天气相关卡
3. 以上都不命中 → 情境匹配分最高的新卡

### 4.6 前 5 张卡的特殊编排（冷启动）

无画像数据时不依赖推荐引擎，人工编排：

| 顺序 | 卡片要求 | 目的 |
|------|---------|------|
| 第 1 张 | 普世微粒卡，任何人都能共鸣，intensity ≤ 0.2 | 破冰 |
| 第 2-5 张 | 分别覆盖 explore / connect / observe / reflect / taste 中的 5 个动作原子，全部低强度 | 采集初始信号 |
| 第 5 张之后 | 底部浮现一行字 2 秒：「你好像更喜欢 __ 类的体验。」 | 建立信任 |

第 5 张之后切换到推荐引擎排序。

## 五、打卡流

### 触发方式

两种入口：
1. 星图中点击暗星 → 选择「我做了」
2. 卡片流中再次遇到已右滑的卡片 → 识别为已存在的暗星 → 提示可打卡

### 打卡界面

```
┌─────────────────────────┐
│                         │
│   [卡片邀请文字]         │
│   你做到了。             │
│                         │
│   选一个感受：           │
│                         │
│   治愈  开心  释然       │
│   疲惫  突破  平淡       │
│                         │
│   ─────────────────     │
│   [焦点追问]             │
│   「灯光是什么颜色？」    │
│                         │
│   [输入框]               │
│                         │
│        [ 完成 ]          │
│                         │
└─────────────────────────┘
```

### 焦点追问

每张卡片在编辑时预设一个焦点问题（观察点）。

| 卡片 | 焦点追问 |
|------|---------|
| 去新餐厅不看菜单 | 「你被推荐了什么菜？」 |
| 换一条路回家 | 「路上有什么你没见过的东西？」 |
| 今晚不带手机上床 | 「第二天醒来第一个念头是什么？」 |
| 给很久没联系的人发消息 | 「对方怎么回的？」 |

焦点追问是可选的。用户可以直接点完成跳过。但一个具体问题比"写一句话"容易回答 10 倍。

### 打卡完成后

1. 屏幕切到星图视角
2. 暗星亮起，颜色 = 情绪对应的氛围色
3. 手机轻震（`navigator.vibrate(50)`）
4. 如果附近有其他亮星（共享动作原子），连线出现
5. 底部浮现情绪回应文案，停留 2-3 秒：

| 情绪 | 回应 |
|------|------|
| 治愈 | 有些事做完就会觉得，本来就该这样。 |
| 开心 | 对吧。 |
| 释然 | 有些重量，放下的时候才发现它一直在。 |
| 疲惫 | 累了就是真的做了。休息一下。 |
| 突破 | 第一次做的事，你永远记得当时的感觉。 |
| 平淡 | 不是每次都有大感觉。但你去做了。 |

6. 2 秒后底部浮现：「还想看看别的吗？」→ 点击回到卡片流

## 六、星图（天空）

### 入口

右上角常驻一个小圆点。随着星星增多，光晕增强。

### 星图界面

```
┌─────────────────────────┐
│  ·          *     ·     │
│        ·              · │
│    ★──────★     ·       │
│     \    /        ·     │
│      ★──★    ·          │
│  ·         ·     ★      │
│       ·       ·         │
│  ·        ○         ·   │
│        ·       ·        │
│  ·   ·     ·        ·   │
│                         │
│  你的天空  ·  7 颗星      │
│                         │
└─────────────────────────┘

★ = 亮星（已完成），颜色 = 打卡情绪色
○ = 暗星（想做还没做），微弱白色
· = 远处微弱光（他人的星，v0.1 用随机生成模拟）
── = 连线（共享动作原子的亮星之间）
```

### 交互

| 操作 | 效果 |
|------|------|
| 点击亮星 | 浮层：卡片邀请文字 + 情绪 + 日期 + 焦点回答 |
| 点击暗星 | 浮层：卡片邀请文字 + 「我做了」按钮 |
| 长按远处微光 | 浮现一行字：「有人在一个雨天做了这件事。」松手消失 |
| 双指缩放 | 拉远看更大的天空 |

### 星的位置算法（v0.1 简化版）

不需要复杂布局。按动作原子分区：

```
星图空间分为 7 个松散区域，对应 7 个动作原子。
同一动作原子的星靠近，不同原子的星远离。
具体位置在区域内随机偏移，避免重叠。
连线：同一主动作原子的亮星之间自动连线。
```

### 暗星生命周期

| 时间 | 状态 |
|------|------|
| 右滑后 | 暗星出现，正常亮度 |
| 7 天未打卡 | 开始微微闪烁 |
| 21 天未打卡 | 极暗，几乎融入背景 |
| 随时 | 用户可在星图中找到并打卡 |

暗星永不消失。

### 延迟洞察

| 触发条件 | 文案 |
|---------|------|
| 第 3 颗亮星 | 「三颗星了。这是一个开始。」 |
| 第 1 个星座（3+ 颗同主原子亮星相连） | 「这几颗星连在了一起。你好像特别喜欢 __。」 |
| 第 10 颗亮星 | 「你的天空开始有形状了。」 |
| 某区域明显密集 | 「你的天空有一个最亮的角落：__。」 |

洞察以浮层形式出现在星图中，首次进入星图时展示，看过后不再重复。

## 七、情境感知系统

### 可用信号与获取方式

| 信号 | 获取 | 用途 |
|------|------|------|
| 时刻 + 星期 | `new Date()` | 时段卡片过滤 |
| 天气 | 免费 API（和风天气） | 天气卡片过滤 + 天气反差检测 |
| 近 5 天天气 | 本地缓存 | 天气反差（连阴后放晴等） |
| 节气 | 本地计算 | 节气精选卡 |
| 日出日落 | 经纬度 + 公式 | 日出日落相关卡片时机 |
| 粗略位置 | 系统定位（可选授权） | 在家/在外判断 |

### 情境过滤规则（硬规则）

| 条件 | 过滤 |
|------|------|
| 23:00-06:00 | 排除所有 space ≠ home 的卡片 |
| 暴雨/极端高温 | 排除所有 space = nature/city 的卡片 |
| 工作日 + 9:00-18:00 | 排除 time > 30min 的卡片 |
| 用户在外面 | 排除 space = home 的卡片 |

### 情境加分规则（软规则）

| 条件 | 加分 |
|------|------|
| 天气反差命中（连阴后放晴） | 户外卡 context_score ×1.5 |
| 节气窗口（前后 3 天） | 节气卡 context_score ×2.0 |
| 周五晚/周六 | intensity 上限放宽一档 |
| 周日下午 | 治愈/安静类卡片加分 |
| 用户在外面 + 周末 | explore 类卡片加分 |

## 八、推荐引擎（v0.1 简化版）

### 数据结构

#### 卡片

```
Card {
  id
  invite: string          // 第一层邀请文字
  hook: string            // 第二层预演钩子
  preview: string         // 第三层完整预演
  preview_type: enum      // 叙事/事实/多人声音/问题/挑战/极短
  focus_question: string  // 焦点追问
  voice: string           // 他人痕迹文案
  atmosphere: enum        // 氛围色：warm/blue/green/purple/white

  // 原子标签
  actions: { primary: ActionAtom, secondary?: ActionAtom }
  context: { social, space, time, cost, intensity }
  impact: { vitality?, curiosity?, bond?, ground?, agency?, depth? }

  // 时效
  seasonal?: Season[]
  time_of_day?: TimeSlot[]
  weather?: WeatherCondition[]
  solar_term?: SolarTerm[]
}
```

#### 用户画像

```
UserProfile {
  P: number[7]            // 偏好向量，初始全 0
  S: number[6]            // 状态向量，初始全 0
  checkin_count: number   // 累计打卡数
  history: Event[]        // 行为日志
}
```

### 推荐流水线

```
全量卡片（30-50 张）
    │
    ▼
① 硬过滤：去掉已完成 / 已拒绝 / 3 天内展示过
    │
    ▼
② 情境硬过滤：去掉当前时间/天气/位置不可能的
    │
    ▼
③ 打分排序：
   score = context_score × 0.4 + profile_score × 0.4 + novelty × 0.2
    │
    ▼
④ 策略调整：
   - 同主原子最多连续 2 张
   - 每 5 张插 1 张野卡（profile_score 最低 20% 中随机）
   - 强度渐进（打卡 <3 次只推 intensity ≤ 0.3）
    │
    ▼
⑤ 精选卡插入首位
    │
    ▼
卡片栈
```

### 画像更新规则

| 行为 | P 更新 | S 更新 |
|------|--------|--------|
| 右滑 | 主原子 +0.15，次原子 +0.08 | 无 |
| 左滑-不感兴趣 | 主原子 -0.10 | 无 |
| 左滑-现在不合适 | 无 | 无 |
| 展开预演 | 主原子 +0.05 | 无 |
| 打卡完成 | 主原子 +0.5，次原子 +0.25 | impact 对应维度 +1.0 |
| 打卡情绪正向（治愈/开心/突破） | 额外 +0.2 | 额外 +0.15 |

每日衰减：`P[i] *= 0.98`，`S[i] *= 0.977`

## 九、卡片库要求（v0.1）

### 数量：30-50 张

### 分布要求

**按动作原子**：

| 原子 | 数量 | 占比 |
|------|------|------|
| explore | 7-8 | ~18% |
| taste | 5-6 | ~13% |
| create | 4-5 | ~10% |
| connect | 5-6 | ~13% |
| move | 4-5 | ~10% |
| observe | 6-7 | ~15% |
| reflect | 6-7 | ~15% |

**按强度**：

| 强度 | 占比 |
|------|------|
| 低（intensity ≤ 0.3） | 40% |
| 中（0.3 < intensity ≤ 0.6） | 40% |
| 高（intensity > 0.6） | 20% |

**按调性**：

| 调性 | 占比 |
|------|------|
| 安静内省 | 25% |
| 日常冒险 | 25% |
| 有趣/幽默 | 20% |
| 直接挑战 | 15% |
| 实用 | 15% |

**特殊卡片**：
- 节气卡：3-5 张（覆盖最近 2 个月的节气）
- 天气卡：3-4 张（晴天专属、雨天专属、降温专属）

### 每张卡片必须包含

- [ ] 邀请文字（画面感，非标题）
- [ ] 预演钩子（最有力的 1-2 句）
- [ ] 完整预演（6 种形态选 1）
- [ ] 焦点追问（1 个具体问题）
- [ ] 他人痕迹文案（1 条模拟声音）
- [ ] 原子标签（动作 + 情境 + 画像影响）
- [ ] 氛围色
- [ ] 时效标签（如有）

## 十一、核心体验流完整走查

```
首次打开
```

## 十二、技术选型（建议）

| 层面 | 选择 | 理由 |
|------|------|------|
| 框架 | React + Vite | 你熟悉，启动快 |
| 状态管理 | Zustand | 轻量，persist 中间件直接做本地持久化 |
| 动画 | Framer Motion | 卡片滑动 + 星图动画 |
| 星图渲染 | Canvas 或 SVG | 星图交互不复杂，SVG 先行，性能不够再换 Canvas |
| 天气 API | 和风天气免费版 | 国内可用，免费额度足够 |
| 部署 | Vercel | 免费，分享链接即可 |
| 持久化 | localStorage | v0.1 不需要云端 |
