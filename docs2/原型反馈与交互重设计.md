## 一、星空是同一片星空——机位不同，空间唯一

### 核心原则

**整个 App 里只有一片星空。** 卡片流、星图全景、着陆、打卡——所有界面看到的都是同一片星空的不同机位。星星的位置、连线、亮度在任何视角下都是一致的。

### 机位系统

想象一个摄影机对着一片星空：

| 界面 | 机位 | 星空表现 | 过渡动效 |
|------|------|---------|---------|
| 着陆 | 仰望 45° | 全屏可见，星空清晰，文字浮在中间 | — |
| 卡片流 | 仰望 60°（稍微低头看卡片） | 全屏背景，上方清晰、下方被卡片遮挡 | 从着陆：星空不变，文案淡出，卡片从下方升入 |
| 星图全景 | 正上方 90°（仰望） | 全屏、最清晰、所有星可交互 | 从卡片流：卡片向下沉出画面，星空亮度提升，用户星星放大变清晰 |
| 打卡完成 | 推近特写 | 聚焦到刚亮的那颗星附近 | 从打卡：背景过渡到星图，镜头推近到目标星 |

### 实现方式

星空是一个**单一的全局 Canvas/SVG 层**，始终渲染。不同界面通过以下参数控制：

```
starFieldState = {
  opacity: 0-1,        // 整体亮度
  scale: 0.5-2.0,      // 缩放（远景→特写）
  offsetY: -200~200,   // 垂直平移
  userStarsOpacity: 0-1 // 用户星星（亮星/暗星/连线）的可见度
}
```

| 界面 | opacity | scale | offsetY | userStarsOpacity |
|------|---------|-------|---------|-----------------|
| 着陆 | 0.7 | 0.8 | 0 | 0.6 |
| 卡片流 | 0.4 | 0.8 | -60 | 0.3 |
| 星图全景 | 0.2（背景星暗）| 1.0 | 0 | 1.0 |
| 打卡特写 | 0.15 | 1.8 | 聚焦到目标星 | 1.0 |

界面切换时，这四个参数用 spring 动画过渡。用户看到的就是**同一片天空在平滑变焦**。

### 点击星空区域进入全景

在卡片流中，卡片上方的空白区域（星空可见区）是可交互的：

- **点击星空区域**：进入星图全景（等同于点右上角呼吸点）
- **从星图全景下拉或点击底部**：卡片从下方升回，退出全景

不需要额外按钮。星空本身就是入口。

## 二、手势引导——不激进，但克制

### 问题

完全没有引导 → 用户茫然。全靠箭头 → 莫名其妙。

### 方案：首次 3 张卡有"示意动效"，之后消失

#### 第 1 张卡片

卡片出现后静止 1.5 秒，然后**卡片自己微微向右上方漂移 6px 再弹回**。像在说"你可以往这个方向推我"。

同时卡片上方出现极淡的文字（opacity 0.2），2 秒后淡出：

> 向上或向右 · 想做

这是唯一一次出现文字引导。

#### 第 2 张卡片

只有微漂移示意，没有文字。

#### 第 3 张及之后

无任何引导。用户已经懂了。

#### 翻转引导

第 1 张卡片如果用户 5 秒内没有点击，卡片微微抖动一下（像有东西在背面）。暗示"你可以翻我看看"。

只做一次。

## 三、拖拽方向反馈——卡片有"生命感"

### 现在的问题

四方向都能拖，但视觉反馈不够，用户不知道什么会发生。

### 飞升方向（右/上）的反馈层次

拖动距离分 3 档：

| 阶段 | 距离 | 视觉反馈 |
|------|------|---------|
| 试探 | 0-30px | 卡片边框开始微微发光（氛围色） |
| 确认 | 30-80px | 卡片整体变亮 + 边框光增强 + 卡片开始向上微微缩小（形变：从矩形向光点过渡的开始）+ 星空背景轻微变亮 |
| 触发 | >80px | 松手触发飞升动画。卡片加速缩小 → 变成一个光点 → 飞入星空中对应位置成为暗星 |

飞升触发后的动画细节（400ms）：
```
0ms   卡片开始缩小 + 圆角增大（20→50%）
100ms 卡片变成小光球，氛围色发光
200ms 光球加速飞向星空中的目标位置
350ms 光球到达位置，扩散一圈微光
400ms 暗星稳定出现
```

**同时**，星空区域浮现一行字（由卡片的 space 决定）：

| space | 飞升后文案 |
|-------|-----------|
| home | 「今晚试试。」 |
| nearby | 「下次路过。」 |
| city | 「找个时间。」 |
| nature | 「等一个好天气。」 |

1.5 秒淡出。这就是"实施意图的种子"。

### 沉降方向（左/下）的反馈层次

| 阶段 | 距离 | 视觉反馈 |
|------|------|---------|
| 试探 | 0-30px | 卡片开始变暗（降低 brightness） |
| 确认 | 30-80px | 卡片变得半透明 + 微微模糊 + 整体下沉感（Y 轴微缩） |
| 触发 | >80px | 松手触发沉降动画。卡片继续下沉 + 淡出消失 |

沉降动画（300ms）：
```
0ms   卡片加速下沉/左移
100ms 透明度降到 0.3
200ms 模糊增强，像沉入水底
300ms 完全消失
```

沉降后没有文案。安静地消失。

### 反馈的"生命感"

关键在于：卡片在被拖动时**像一个有质感的物体在回应你**。

飞升方向 = 卡片在发光，在"期待"被选中
沉降方向 = 卡片在暗淡，在"接受"被放下

这种拟物的情感暗示比任何文字引导都有效。

## 四、星星形态——不是圆点，是星

### 问题

纯圆形 → 看不出是星星。

### 设计

星星用 **四角星形 + 光晕** 的组合。

#### 亮星

```
     ·
   · · ·
  ·  ★  ·     ← 四角星芯（实心，情绪色）
   · · ·       ← 十字光芒（半透明延伸）
     ·          ← 外圈柔和光晕（radial-gradient）
```

实现方式（SVG/Canvas）：
1. **核心**：一个 4 角星形 path（不是圆），大小根据卡片强度 3-6px
2. **十字光芒**：从中心向四个方向延伸的细线，长度 = 核心的 2-3 倍，半透明
3. **光晕**：radial-gradient 圆形光晕，颜色 = 情绪色，半径 = 核心的 4-5 倍

不同状态：

| 状态 | 核心 | 光芒 | 光晕 | 动效 |
|------|------|------|------|------|
| 亮星 | 实心，情绪色 | 明显 | 柔和扩散 | 稳定微闪 |
| 暗星 | 半透明白 | 几乎不可见 | 极淡 | 缓慢呼吸明灭 |
| 暗星 >7天 | 更暗 | 无 | 无 | 不规则闪烁（像快要熄灭） |
| 刚点亮的星 | 从暗→亮的爆发 | 从无→展开 | 脉冲扩散 | 点亮动画 1.2s |

#### 背景星（装饰）

保持简单的小圆点。和用户星星有明显区分——背景星小且暗，用户星星有形态有光芒。

## 五、信笺系统——融入卡片流

### 信笺的视觉

信笺和体验卡片共享同一个"浮动卡片"的形态，但有明确的视觉区分：

```
┌──────────────────────────────────┐
│                                  │
│  [无氛围光，极淡白色边框光]       │
│                                  │
│                                  │
│     你最近选的都和                │
│     「探索」有关。                │
│                                  │
│                                  │
│                                  │
└──────────────────────────────────┘
```

区别：
- 没有氛围色光晕（或用极淡的白光）
- 边框是虚线或更淡的实线
- 字号稍小，居中
- 背景更透明（0.7 而不是 0.88）

### 信笺的交互

信笺不能飞升/沉降（没有意义——它不是体验）。

- **纯信息类**（画像确认、里程碑、预测）：向任意方向轻推即消失
- **有选择的**（画像提问、回访）：展示 2-3 个选项按钮，选择后消失
- **实施意图引导**：展示暗星列表，点击某颗暗星后信笺消失

### 出现时机（v0.1 先实现以下 3 种）

| 类型 | 触发 | 内容示例 |
|------|------|---------|
| 画像确认 | 累计 5/10/15 次选择（飞升+沉降） | 「你好像更喜欢 __ 类的体验。」 |
| 精选预告 | 每天第一张卡之前 | 「今天为你选了一张。」 |
| 里程碑 | 第 1/3/5/10 次打卡 | 「你的天空有 N 颗星了。」 |

### 在调试面板中触发

调试面板增加：
- 「插入信笺：画像确认」
- 「插入信笺：精选预告」
- 「插入信笺：里程碑」

点击后在卡片流中立刻出现对应信笺。

## 六、精选卡

### 视觉

精选卡和普通卡共享同一个形态，但有一个独特的视觉特征：

- **边框光更亮**：从 `rgba(255,255,255,0.08)` 提升到 `rgba(255,255,255,0.2)`
- **氛围光有脉动**：以 4 秒为周期，氛围光的强度在 0.6-1.0 之间缓慢波动
- **卡片顶部**有一个极小的标记：一颗微缩的四角星 ✦，表示"这张是精选的"

### 逻辑

- 每天的第一张卡是精选
- 优先级：节气卡 > 天气反差卡 > 情境匹配最高分的新卡

### 调试面板

增加：「下一张设为精选」

## 七、沉降卡片的找回——确认可用

星图全景模式的底部，当有沉降卡片时：

```
                    ↑ 长按看详情

    ★      ·    ○     ★──★
         ·         ·
      ·       ★         ·
   ·      ·        ·

─────────────────────────────
  还有 3 张你放下的          ←  点击展开
```

点击后从底部滑出一个半高面板（sheet）：

```
┌──────────────────────────────────┐
│        你放下的                   │
│                                  │
│  [暖橙·] 坐下来，告诉服务员…     → 找回 │
│  [绿色·] 去超市，只逛从来不逛…   → 找回 │
│  [紫色·] 24小时不打开朋友圈…     → 找回 │
│                                  │
└──────────────────────────────────┘
```

每行左侧一个氛围色小点，右侧"找回"可点击。点击后这张卡变成暗星出现在星图中，面板自动关闭。

## 九、保留的设计（无变化）

以下设计从 v1.0 保留，无修改：

- **四方向手势**：右/上=飞升，左/下=沉降，点击=翻转
- **卡片毛玻璃材质**：backdrop-filter + 半透明 + 微边框
- **三层架构**：星空层 / 卡片层 / 浮层
- **手势隔离**：卡片区拖动不影响星空，双指缩放只作用于星空
- **打卡流**：选情绪 → 焦点追问 → 点亮 → 星图特写 → 自然回到卡片流
- **暗星生命周期**：7 天开始闪烁，21 天极暗
- **去掉客气话**：无"还想看看别的吗"、无"你做到了"
- **情绪回应文案**：打卡后在星图中浮现 2 秒
- **延迟洞察**：第 3/5/10 次打卡触发
