# 原型反馈与交互重设计

> 基于体验原型后的问题逐一回应、深入思考、给出设计方案。

## 一、星空作为全局背景层——不是"一个页面"，是"永远在那里"

### 你的反馈

着陆屏的"此刻状态"和星空意象脱节。星空应该一直在，不是藏在右上角。

### 重新设计

**星空不是一个页面，是整个 App 的底层。**

把屏幕想象成一个连续的星空空间，卡片浮在其中：

```
┌─────────────────────────┐
│                         │
│   ★  ·    ★    ·  ★     │  ← 星空（永远在，全屏）
│  ·    ○    ·  ★──★  ·   │     上方更密集、更清晰
│   ·     ·    ·    ·     │     向下自然渐淡
│                         │
│                         │
│     [ 卡 片 区 域 ]      │  ← 卡片浮在星空之上
│                         │
│                         │
│                         │
└─────────────────────────┘
```

日常状态下：
- 星空铺满全屏作为背景，上方更清晰、下方自然渐淡
- 卡片浮在星空之上，居中偏下
- 星空一直在呼吸般微微闪烁

用户向上拖拽 / 点击星空区域 / 点右上角呼吸点：
- 星空层"拉下来"占满全屏 = 进入星图全景
- 卡片区域向下推出视口
- 是**同一个空间的镜头移动**，不是页面跳转

### 星空的镜头语言

| 界面状态 | 星空镜头 | 星空占比 | 效果 |
|---------|---------|---------|------|
| 卡片流（日常） | 远景/仰望 | 全屏背景，上密下淡 | 你在星空中，卡片浮在眼前 |
| 星图全景 | 全景/俯瞰 | 100% | 你飞到了天上，看到全部 |
| 打卡完成瞬间 | 特写 | 60-70% | 镜头推近到那颗刚亮的星 |
| 洞察浮现 | 中景 | 50% | 某个星座区域高亮 |

镜头之间用平滑缩放过渡，没有硬切。用户感知到的是**一个连续的空间在变换视角**。

### "此刻状态"融入星空

不再需要单独的着陆屏。打开 App 时：

星空在上方微微呼吸。下方卡片区域还没出现，取而代之的是一行文字浮在星空下方——这行字就是"此刻状态"：

```
┌─────────────────────────┐
│   ★  ·    ★    ·        │
│  ·    ○    ·  ★──★  ·   │
│   ·     ·    ·    ·     │
│                         │
│                         │
│   周五晚上。天气晴。      │  ← 情境感知文案
│   你有一颗星还没亮。      │  ← 状态文案
│                         │
│       ↑ 上滑看看          │  ← 自然引导
│                         │
└─────────────────────────┘
```

上滑 → 卡片从下方升起，此刻文案淡出，进入卡片流。

**情境感知文案的设计**（替代"看看今天有什么"）：

| 情境 | 文案 |
|------|------|
| 周五晚上 + 晴 | 「周五晚上。天气刚好。」 |
| 下雨 | 「下着雨。适合在家做点什么。」 |
| 连续 5 天没打开 | 「好久不见。你的天空还在。」 |
| 有暗星 >7 天 | 「有一颗星等了你一周了。」 |
| 上次打卡是 3 天前 | 「上次你换了一条路回家，觉得治愈。」 |
| 用户第 1 次打开 | 不展示此刻状态，直接从卡片开始 |

文案互斥，按优先级取最有信息量的一条。简短、具体、有感知。

### 星的数量展示

不主动展示数字。而是在星图全景中，当用户**长按某个星座区域**时，浮现：

> 「探索 · 3 颗亮星 · 1 颗暗星」

松手消失。是用户主动探查到的信息，不是塞给他的数据。

未来状态分类可以扩展（比如"回忆星"= 打卡后 14 天的回访标记），但交互范式不变：都是长按探查。

## 二、手势系统重设计——四方向 + 点击

### 你的反馈

右滑/上滑都应该是"选择"（飞升到星空），左滑/下滑是"沉降"。底部按钮指引困惑。1.5 秒消失的选项很差。

### 重新设计

**去掉所有按钮和文字指引。纯手势。**

```
             ↑ 上
             飞升

    ← 左          右 →
    沉降           飞升

             ↓ 下
             沉降
```

#### 飞升（右滑 / 上滑）= 想做

卡片向右上方飞出，缩小为一颗光点，融入上方星空 → 成为一颗暗星。

视觉：卡片 → 光粒子化 → 星空中多了一颗暗星。整个过程 400ms。

#### 沉降（左滑 / 下滑）= 现在不要

卡片向下沉，逐渐透明，沉入"地面"以下消失。

**沉降的概念：不是拒绝，是"沉入地下"。**

沉降的卡片不消失。它们沉入"地层"——一个看不见的、安静的地方。如果说星空是"你想做和做过的事"，地层就是"你见过但放下了的事"。

用户不需要在沉降时做任何选择（不需要区分"不感兴趣"和"现在不合适"）。系统从行为模式中自然推断：

- 看了 <1 秒就沉降 = 大概率不感兴趣
- 看了 >3 秒甚至展开过预演再沉降 = 可能只是时机不对

这比 1.5 秒弹窗好 100 倍。因为**时间本身就是信号**。

#### 沉降卡片的找回（v0.1 包含）

沉降的卡片不是消失了，是"暂时离开了视野"。

在星图全景中，底部有一个微弱的入口——向上轻轻浮现的一行小字：

> 「还有 N 张你见过的」

点击 → 展开沉降卡片列表（极简，每行一句邀请文字 + 淡氛围色），点击任意一张 → 可以重新飞升为暗星。

**概念上：星空是你选择的和做过的。沉降区是你见过但放下的。都在同一片宇宙里，只是距离不同。** 没有"地面/地层"的概念，一切都在星空的维度里。

#### 点击 = 翻转/展开

点击卡片 → 卡片翻转（或扩展）→ 展示预演详情面。再点一次 → 翻回正面。

**展开状态下仍然可以飞升/沉降。** 这是关键——不能因为展开了就锁死交互。展开状态就是多看了一眼，看完之后照样可以滑动决策。

#### 手势反馈

拖动过程中的视觉反馈：

| 拖动方向 | 实时反馈 |
|---------|---------|
| 向右/上拖 | 卡片开始发光，越拖越亮，星空背景微微变亮 |
| 向左/下拖 | 卡片开始变暗，越拖越透明，微微下沉 |
| 松手不够远 | 卡片弹回原位 |

阈值：拖动超过屏幕宽度/高度的 30% 触发飞升/沉降。

## 三、卡片视觉与质感

### 你的反馈

透明度太高导致重叠，屏幕太空，缺少高级质感。

### 重新设计

#### 卡片材质

```
┌──────────────────────────────────┐
│ ┌──────────────────────────────┐ │
│ │                              │ │
│ │   [氛围色光晕，模糊扩散]      │ │
│ │                              │ │
│ │                              │ │
│ │     今晚上床的时候，          │ │
│ │     把手机放在                │ │
│ │     够不到的地方。            │ │
│ │                              │ │
│ │                              │ │
│ │     头十分钟手会有点空。      │ │
│ │     第二天醒来，第一件事      │ │
│ │     是自己的意识，不是通知。   │ │
│ │                              │ │
│ │                              │ │
│ └──────────────────────────────┘ │
│              · · ·               │  ← 卡片指示点（当前/总数）
└──────────────────────────────────┘
```

视觉要素：
- **背景**：不是纯透明。用 `backdrop-filter: blur(20px)` + 半透明深色底（rgba(15,20,25,0.85)）。毛玻璃质感，能看到星空透过来但不会重叠文字
- **边框**：极细的 1px 亮边（rgba(255,255,255,0.08)），给卡片一个微微的"存在感"
- **氛围光**：卡片内部上方有一团模糊的氛围色光晕（和星空中这颗星的颜色一致），不是纯色背景，是**光**
- **阴影**：向下的柔和阴影（`0 20px 60px rgba(0,0,0,0.4)`），让卡片"浮"在星空之上
- **圆角**：20px，统一

#### 只展示当前一张卡片

不要卡片堆叠。**当前只有一张卡片在屏幕上。** 下一张在飞升/沉降动画完成后才出现（从下方淡入升起）。

这解决了透明度重叠问题，也让每张卡片有独占的注意力。

#### 屏幕"太空"的解决

屏幕不空——**星空就是内容**。卡片占据下方 60%，上方 40% 是活的星空。

让星空本身有呼吸感：
- 星星微微闪烁（不同频率）
- 偶尔有流星划过（纯装饰，低频）
- 暗星比亮星暗但能看到
- 星座连线若有若无

当用户在卡片流中静止不动 >5 秒，星空的亮度微微提升——像在说"抬头看看"。

## 四、卡片翻转——预演作为卡片背面

### 你的反馈

预演做成点击翻转。少按钮，宽触发区域。

### 设计

点击卡片任意位置 → 卡片 3D 翻转到背面（Y 轴旋转 180°，300ms）。

**正面**（日常看到的）：
```
邀请文字
+
预演钩子
```

**背面**（点击后翻转）：
```
完整预演

──────

约 1-2 小时 · ¥40-120
城市 · 独自或结伴

──────

「第一次被推荐了一道
  我绝对不会自己点的菜。」
```

背面仍可飞升/沉降。再点一次翻回正面。

翻转的好处：
1. 无按钮，纯手势
2. 正面极简（两层文字），背面丰富（全部信息）
3. 翻转本身有"探索感"——你在翻看这张卡片
4. 触发区域 = 整张卡片 = 最大化

## 五、"信笺"——卡片间插入的系统对话

### 你的反馈

初次 5 张后的预判/交互应该长期融入卡片流。可以是预测、画像细化问题、状态确认等。

### 新概念：信笺（Note）

在卡片流中，偶尔插入一种特殊的卡片——**信笺**。它不是体验卡片，是系统和用户之间的一次轻对话。

视觉上和体验卡片有区分：
- 没有氛围光（或用更淡的白色光）
- 文字更小、更居中
- 不能飞升/沉降，只能向下轻轻推走（确认已读）

#### 信笺类型

| 类型 | 触发条件 | 内容 | 交互 |
|------|---------|------|------|
| **画像确认** | 每 5 次选择后 | 「你最近选的都和"探索"有关。」 | 轻推消失 |
| **画像提问** | 某维度信号矛盾时 | 「你想做社交类的事但都没去做——是不想，还是没机会？」→ [不想] [没机会] | 选择后消失 |
| **预测** | 推荐引擎高置信度时 | 「接下来这张，我觉得你会喜欢。」 | 轻推消失，然后看到那张卡 |
| **回访** | 某次打卡后 14 天 | 「两周前你第一次不看菜单点菜。现在回想——」→ [值得] [一般] [想再来一次] | 选择后消失 |
| **实施意图引导** | 用户有 3+ 颗暗星 | 「你想做的事里，哪件最快能做？」→ 展示暗星列表，点击任一颗 | 选中后暗星高亮 |
| **里程碑** | 第 3/5/10 次打卡 | 「你的天空有 5 颗星了。」 | 轻推消失 |

#### 出现频率

**不能太多。** 信笺和卡片的比例约 1:8。即每 8 张体验卡片中穿插 1 张信笺。

信笺的存在让卡片流不再只是"刷刷刷"，而是有节奏的——大部分是体验卡片，偶尔系统和你说一句话。像一个安静的朋友偶尔插一句嘴。

## 六、实施意图——不用时间绑定的方式

### 你的反馈

实施意图在原型没体现，要做交互设计。

### 之前否决了"什么时候做"的时间绑定。用什么替代？

**用空间绑定替代时间绑定。**

人的行动不是被闹钟触发的，是被场景触发的。"下班路过那家店的时候"比"周三晚上 7 点"更可能让你行动。

#### 交互设计

当用户飞升一张卡片后，如果这张卡片的 `space ≠ home`，星空中这颗新暗星会微微闪一下，然后在星空区域浮现一行字（2 秒后淡出）：

> 「下次路过的时候。」

不问"什么时候"，不问"在哪里"。就是一个轻轻的心理锚定。

对于 `space = home` 的卡片：

> 「今晚试试。」

极简。不是计划，是一个念头的种子。

#### 进阶（v0.2+）：位置感知提醒

如果用户授权了位置，当用户到达某颗暗星对应的 space 类型附近时（比如到了城市商圈 = city 类暗星），可以在下次打开 App 时的此刻文案中说：

> 「你现在在外面。有一颗星说'走进一家你路过很多次但从没进去的店'——今天？」

不是推送通知。是你打开 App 时发现它知道你在哪。

## 七、原型中缺失的 MVP 概念

### 精选卡

视觉区分：精选卡的氛围光更亮，有微微的脉动动画（像恒星的呼吸）。

卡片正面额外标识（不是文字标签，是视觉语言）：
- 氛围光的脉动频率更慢、更强
- 或者卡片边框光亮度更高

系统每天选一张精选。打开 App 后的第一张就是它。

### 星星的各异

| 属性 | 视觉差异 |
|------|---------|
| 情绪色 | 治愈=蓝绿、开心=暖黄、释然=淡紫、疲惫=暗橙、突破=亮红、平淡=白 |
| 大小 | 微粒卡=小星、中卡=中星、大卡=大星 |
| 亮度 | 暗星 30% 亮度 / 亮星 100% / 7 天未动的暗星 15% 并闪烁 |
| 形态 | 亮星稳定发光 / 暗星微微呼吸式明灭 |

### 焦点追问

打卡流程中，情绪选择后展示焦点问题：

```
你做到了。

[治愈] [开心] [释然]
[疲惫] [突破] [平淡]
         ↓ 选了"治愈"

──────

「灯光是什么颜色？」

[输入框，可选跳过]

         [ ★ 点亮 ]
```

点击"点亮"→ 直接切到星图特写，看到暗星亮起。

如果跳过焦点追问也可以，直接点亮。

### 星星的连线

共享主动作原子的亮星之间有连线。

连线样式：
- 很细（0.5px）
- 半透明白色
- 微微发光
- 当两颗星被点击查看时，它们之间的连线短暂变亮

3 颗以上相连 = 形成"星座"，区域有微弱的面积填充（极低透明度）。

## 八、分层交互——星空与前台独立

### 你的反馈

星空缩放时，前台卡片等不应受影响。

### 分层规则

```
┌─ 层级 ─────────────────────────┐
│                                │
│  Layer 3: 浮层（打卡、调试面板）  │  ← 最上层，遮挡一切
│  Layer 2: 卡片/信笺             │  ← 中间层，手势归自己
│  Layer 1: 星空                  │  ← 底层，手势归自己
│                                │
└────────────────────────────────┘
```

**手势隔离**：
- 当手指在 Layer 2（卡片区域）操作时：卡片响应，星空不动
- 当手指在 Layer 1（星空区域，即屏幕上方无卡片的部分）操作时：星空响应，卡片不动
- 双指缩放：只作用于星空层，无论手指在哪（因为卡片没有缩放需求）
- 浮层出现时：Layer 1 和 Layer 2 都不响应手势

### 星图全景模式

从卡片流进入星图全景（上方星空区域点击/上拉）：
- 星空层展开到全屏
- 卡片层下推出屏幕
- 此时全屏都是 Layer 1 的交互区域

从星图退出（下拉）：
- 卡片层从下方升回
- 星空缩回上方 30%

## 九、调试面板

可拖动的半透明圆形 icon，悬浮在右下角。

点击展开二级面板：

```
┌──────────────────┐
│   调试面板        │
│                  │
│  [重置全部数据]   │
│  [跳到第 N 张卡]  │
│  [模拟打卡 3 次]  │
│  [切换天气]       │
│  [切换时段]       │
│  [显示画像向量]   │
│  [显示推荐分数]   │
│                  │
│       [关闭]      │
└──────────────────┘
```

面板本身可拖动、半透明、不遮挡主交互。

## 十、去掉所有"客气话"

| 删掉 | 原因 |
|------|------|
| 「看看今天有什么」 | 替换为情境感知文案 |
| 「还想看看别的吗？」 | 打卡后直接切回星空特写，自然过渡 |
| 「跳过 / 想做」底部指引 | 纯手势，无文字。第一张卡可以有极淡的方向箭头暗示，之后不再出现 |
| 「你做到了。」 | 不需要系统确认你做到了。星亮了就是确认。 |

**原则：用视觉和动效说话，不用文字。** 星亮了 = 你做到了。卡片飞走了 = 你选了。不需要任何文字翻译这些动作的含义。

## 十一、打卡完成后的流程修正

旧：打卡 → 情绪回应文案 → 「还想看看别的吗」→ 回到卡片流

新：

```
打卡 → 选情绪 → 焦点追问（可选）→ 点亮
                                      │
                                      ▼
                          星图特写：那颗星亮起
                          连线出现（如有）
                          情绪回应文案浮现 2 秒
                                      │
                                      ▼
                          自然过渡到卡片流：
                          星图缩回上方 30%
                          下一张卡从下方升起
```
